# IAVM 0.0.7 Delivery Note

**Version:** `0.0.7`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.0.7` 聚焦“async/platform 基线”，在 `0.0.6` 的语义扩展与资源模型基础上，完成了以下目标：

- `async function` / `await` 进入 IAVM 稳定执行路径
- Promise 拥有 `pending/resolved/rejected` 三态
- VM 具备最小 `suspend/resume` 机制，并可恢复 `host.poll` 挂起
- network handle 已覆盖 client/server 两条可验证回归路径

## 2. 主要交付物

### 2.1 代码

- `OpAwait`、`module.Function.Async`、Promise runtime 状态机
- `Suspension` / `ResumeSuspension()` 挂起恢复骨架
- `host.poll` Promise 化，以及基于 handle 的恢复执行
- network handle `dial/listen/accept/send/recv/close` 的 CLI 可运行路径

### 2.2 测试

- runtime async / pending / resume 回归
- integration `network.dial/send/recv` 端到端回归
- integration `network.listen/accept/send/recv` 端到端回归
- CLI `run-iavm --cap-config` 的 network client/server smoke 回归

### 2.3 文档

- `docs/iavm/abi-capabilities.md`
- `docs/plans/2026-04-23-iavm-0.0.7-async-platform-plan.md`
- 本交付说明


## 4. 已知边界

以下内容仍不属于 `0.0.7` 的完整目标范围，应进入后续版本：

- 宿主主动 wakeup/通知机制
- 更真实的 backpressure / 调度 / 事件循环语义
- Promise 与外部异步源之间更细粒度的 continuation 编排

## 5. 版本结论

`0.0.7` 已从“语法上识别 async”推进到“具备可验证的 async/platform 最小闭环”。后续版本可以在不推翻现有 ABI 的前提下，继续把 wakeup 与 backpressure 做深。
