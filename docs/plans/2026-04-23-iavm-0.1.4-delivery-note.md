# IAVM 0.1.4 Delivery Note

**Version:** `0.1.4`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.4` 聚焦“backoff curve profile”，在 `0.1.3` retry/backoff 最小策略基础上完成了以下目标：

- `RetryMultiplier` / `RetryMaxBackoff` 已进入 runtime Options
- capability config 已可覆盖默认 multiplier / max cap
- pending promise 会保留触发时的 backoff curve profile，并在恢复路径中继续生效
- runtime 已明确当前 backoff 为确定性退避，不引入随机 jitter

## 2. 主要交付物

### 2.1 代码

- `runtime.Options.RetryMultiplier`
- `runtime.Options.RetryMaxBackoff`
- backoff 曲线计算 helper
- capability backoff profile 读取与 promise 透传

### 2.2 测试

- backoff 计算回归
- capability backoff profile 覆盖回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

## 4. 已知边界

以下内容仍不属于 `0.1.4` 的完整目标范围，应进入后续版本：

- 随机 jitter
- 更复杂的 backoff 算法族
- backoff 与 circuit breaker 的联动策略

## 5. 版本结论

`0.1.4` 已让 IAVM 的 retry/backoff 从“固定间隔”推进到“可配置退避曲线”，为后续更高级的容错策略打下基础。
