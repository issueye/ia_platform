# IAVM 0.1.16 HTTP Method Exclusion Plan

**Goal:** 在 `0.1.15` 已支持 HTTP status/status-class exclusion 之后，为 `network.http_fetch` 增加 method exclusion，让 HTTP retry policy 的方法维度也具备完整的 inclusion/exclusion 组合能力。

**当前状态:**
- `0.1.15` 已完整交付，HTTP retry 已支持显式 method allowlist 与 default safe-methods gate
- 当前若要临时屏蔽某个 method 的重试映射，只能回退整个 allowlist/default-safe 配置，粒度仍偏粗

## 1. 版本目标

`0.1.16` 完成三件事：

1. **HTTP method exclusion 配置**
   - 支持 `retry_http_excluded_methods`
   - 支持 `retryHTTPExcludedMethods`
   - 在 method 通过 allowlist/default-safe gate 后，仍可被 exclusion 显式阻断

2. **Host + runtime 闭环**
   - 被排除 method 的响应保持原始 HTTP 结果
   - runtime allowlisted `host.call` 不会对被排除 method 触发重试

3. **测试与文档**
   - 增加 host API 与 runtime 回归
   - 更新 ABI / 计划文档 / delivery note

## 2. Milestones

### Milestone 1：DefaultHost method exclusion

**状态:** 已完成。

**已完成内容:**
- `DefaultHost` 已支持 capability 级 method exclusion 配置
- method exclusion 在 allowlist/default-safe gate 之后、状态码/状态族匹配之前生效
- exclusion 可覆盖同一 method 已被 allowlist/default-safe 放行的情况

### Milestone 2：Host + runtime 闭环

**状态:** 已完成。

**已完成内容:**
- host API method exclusion 回归已补齐
- runtime + DefaultHost 的 exclusion method 不重试回归已补齐
- 被排除 method 保持原始 HTTP 响应与单次尝试

### Milestone 3：测试与文档收口

**状态:** 已完成。

**已完成内容:**
- host API exclusion method 回归已补齐
- runtime 集成回归已补齐
- 全量测试已通过
