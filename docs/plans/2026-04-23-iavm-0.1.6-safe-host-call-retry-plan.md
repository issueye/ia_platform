# IAVM 0.1.6 Safe Host Call Retry Plan

**Goal:** 在 `0.1.5` 已完成 jitter policy 之后，为 IAVM 增加“显式安全”的 `host.call` 重试策略，让只读或幂等 operation 可以复用已有 timeout/backoff/jitter profile。

**当前状态:**
- `0.1.5` 已完整交付，`poll/wait` 已支持 timeout/backoff/jitter profile
- `host.call` 当前虽然共享单次 timeout 配置，但仍默认只执行一次

## 1. 版本目标

`0.1.6` 完成三件事：

1. **Safe host.call retry allowlist**
   - 支持 `Options.RetryCallOps`
   - 未进入 allowlist 的 `host.call` 保持单次执行

2. **Capability override**
   - capability config 支持 `retry_call_ops`
   - capability profile 可覆盖默认 allowlist

3. **测试与文档**
   - 增加 allowlist 生效/未生效回归
   - 更新 ABI / 计划文档 / delivery note

## 2. Milestones

### Milestone 1：Runtime allowlist 支持

**状态:** 已完成。

**已完成内容:**
- runtime 已支持 `RetryCallOps`
- `host.call` 仅在 operation 命中 allowlist 时才进入 retry 路径
- allowlist 命中后可复用既有 retry/backoff/jitter 配置

### Milestone 2：Capability profile 覆盖

**状态:** 已完成。

**已完成内容:**
- capability config 已可读取 `retry_call_ops`
- capability allowlist 可覆盖默认 allowlist
- operation 级 allowlist 判定已接入 `OpHostCall`

### Milestone 3：测试与文档收口

**状态:** 已完成。

**已完成内容:**
- 默认不重试 / allowlist 重试 / capability allowlist 重试回归已补齐
- runtime 与全量测试已通过
- ABI / delivery note 已更新

## 3. 发布前验证

```bash
go test ./iavm/pkg/runtime
go test ./iacommon/... ./iavm/... ./ialang/...
```

## 4. 非目标范围

- 对所有 `host.call` 一刀切自动重试
- 按错误类别区分更多 retry policy
- retry 与 circuit breaker / budget policy 的联动
