# IAVM 0.1.22 Delivery Note

**Version:** `0.1.22`  
**Date:** `2026-04-24`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.22` 聚焦“capability call-retry disable”，为 capability 增加了仅关闭 `host.call` 自动重试的细粒度开关：

- capability 现已支持 `retry_call_enabled`
- `retry_call_enabled = false` 时，该 capability 的 allowlisted `host.call` 不再自动重试
- 同一 capability 的 `poll/wait` retry 行为保持不变

## 2. 主要交付物

### 2.1 代码

- capability call-retry disable 配置解析
- capability 级 host.call retry policy 清零逻辑
- host.call disable 与 poll 保持重试回归测试

### 2.2 测试

- capability host.call retry disable 回归
- capability poll retry 保持生效回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

## 4. 已知边界

以下内容仍不属于 `0.1.22` 的完整目标范围，应进入后续版本：

- 更细粒度的 `retry_poll_enabled` 独立开关
- `host.call` 内部再细分为 sync/async 操作独立开关
- 与 circuit breaker / concurrency policy 的联动

## 5. 版本结论

`0.1.22` 让 IAVM 的 capability retry 开关从“全开/全关”推进到“可单独关闭 host.call retry”，更适合既需要等待重试又不希望调用重放的宿主能力。
