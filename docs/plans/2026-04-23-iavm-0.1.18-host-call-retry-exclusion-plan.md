# IAVM 0.1.18 Host Call Retry Exclusion Plan

**Goal:** 为 runtime 的 `host.call` retry allowlist 增加 exclusion 能力，让调用级重试策略也具备 inclusion/exclusion 组合控制。

**当前状态:**
- `0.1.17` 已完整交付，runtime retry 已支持 max elapsed budget
- 当前 `host.call` 只有 `retry_call_ops` allowlist；若想在 allowlist 基础上临时屏蔽某个 operation，只能整体改写 allowlist

## 1. 版本目标

`0.1.18` 完成三件事：

1. **Host call retry exclusion 配置**
   - 支持 `Options.RetryExcludedCallOps`
   - 支持 capability `retry_excluded_call_ops`
   - exclusion 优先级高于 allowlist 命中

2. **Runtime 闭环**
   - 命中 exclusion 的 `host.call` operation 保持单次尝试
   - exclusion 可同时覆盖默认 allowlist 与 capability allowlist

3. **测试与文档**
   - 增加 runtime allowlist/exclusion 优先级回归
   - 更新 ABI / 计划文档 / delivery note
