# IAVM 0.1.22 Capability Call Retry Disable Plan

**Goal:** 为 capability 增加 `retry_call_enabled` 细粒度开关，让单个 capability 可以只关闭 `host.call` 自动重试，而保留 `poll/wait` retry。

**当前状态:**
- `0.1.21` 已完整交付，capability 已支持 `retry_enabled` 总开关
- 当前若只想保留 `poll/wait` retry、同时关闭 `host.call` retry，仍只能用总开关一起关闭

## 1. 版本目标

`0.1.22` 完成三件事：

1. **Capability call-retry disable 配置**
   - 支持 `Config.retry_call_enabled`
   - 支持 `Config.retryCallEnabled`
   - 为 `false` 时只关闭该 capability 的 `host.call` 自动重试

2. **Host.call / poll 闭环**
   - allowlisted `host.call` 在 capability 级 call-retry disable 下回退为单次尝试
   - 同一 capability 的 `poll/wait` 仍可继续使用 retry/backoff profile

3. **测试与文档**
   - 增加 runtime capability call-retry disable 回归
   - 更新 ABI / 计划文档 / delivery note

## 2. Milestones

### Milestone 1：Capability call-retry disable

**状态:** 已完成。

**已完成内容:**
- capability profile 已支持 `retry_call_enabled`
- 关闭时仅清空 `host.call` retry allow/exclude policy
- 不影响同一 capability 的 poll/wait retry 配置

### Milestone 2：运行时闭环

**状态:** 已完成。

**已完成内容:**
- capability host.call retry disable 回归已补齐
- capability poll retry 保持生效回归已补齐
- `retry_call_enabled = false` 时语义稳定收敛到“call 单次、poll 可重试”

### Milestone 3：文档收口

**状态:** 已完成。

**已完成内容:**
- ABI 文档已补充 capability call-retry disable 开关
- 计划文档与 delivery note 已补齐
