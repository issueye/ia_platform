# IAVM 0.1.13 Default Safe HTTP Methods Plan

**Goal:** 在 `0.1.12` 已完成 HTTP status class 配置之后，为 `network.http_fetch` 增加一个可选的“标准幂等方法默认集”开关，降低配置成本，同时保持现有显式 method guard 语义不变。

**当前状态:**
- `0.1.12` 已完整交付，`network.http_fetch` 已支持显式 status/status-class + method guard
- 当前如果想限制到安全方法，仍需手动写 `retry_http_methods`

## 1. 版本目标

`0.1.13` 完成三件事：

1. **Default safe methods 开关**
   - 支持 `retry_http_default_safe_methods`
   - 支持 `retryHTTPDefaultSafeMethods`
   - 仅在未显式提供 `retry_http_methods` 时生效

2. **Host + runtime 闭环**
   - `GET/DELETE` 等标准幂等方法可直接走重试映射
   - `POST` 等未进入默认集的方法保持原始 HTTP 响应

3. **测试与文档**
   - 增加 host API 与 runtime 回归
   - 更新 ABI / 计划文档 / delivery note

## 2. Milestones

### Milestone 1：DefaultHost 默认安全方法集

**状态:** 已完成。

**已完成内容:**
- `DefaultHost` 已支持默认安全方法集开关
- 标准幂等方法集为 `GET/HEAD/OPTIONS/TRACE/PUT/DELETE`
- 显式 `retry_http_methods` 仍优先于默认集

### Milestone 2：Host + runtime 闭环

**状态:** 已完成。

**已完成内容:**
- `GET/DELETE` 命中默认安全方法集的 host API 回归已补齐
- runtime + DefaultHost 的默认安全方法集回归已补齐
- `POST` 保持原始 HTTP 响应

### Milestone 3：测试与文档收口

**状态:** 已完成。

**已完成内容:**
- host API 默认安全方法集回归已补齐
- runtime 集成回归已补齐
- 全量测试已通过
