# IAVM 0.1.24 Delivery Note

**Version:** `0.1.24`  
**Date:** `2026-04-24`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.24` 聚焦“capability wait-retry disable”，为 capability 增加了仅关闭 `wait(handle)` 自动重试的细粒度开关：

- capability 现已支持 `retry_wait_enabled`
- `retry_wait_enabled = false` 时，该 capability 的 `wait(handle)` 不再自动重试
- 同一 capability 的 `poll` 与 allowlisted `host.call` retry 行为保持不变

## 2. 主要交付物

### 2.1 代码

- capability wait-retry disable 配置解析
- pending promise wait-retry 开关持久化
- wait disable、poll 保持与兼容联动回归测试

### 2.2 测试

- capability wait retry disable 回归

## 4. 已知边界

以下内容仍不属于 `0.1.24` 的完整目标范围，应进入后续版本：

- 更细粒度的 wait/poll 独立 backoff 曲线
- 对不同 waiter 实现配置不同 retry profile
- 与 circuit breaker / concurrency policy 的联动

## 5. 版本结论

`0.1.24` 让 IAVM 的异步等待 retry 开关从“poll/wait 一起开关”进一步细化到“可单独关闭 wait”，并保留了旧版本配置语义的兼容性。
