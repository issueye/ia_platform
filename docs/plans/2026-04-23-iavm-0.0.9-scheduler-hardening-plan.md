# IAVM 0.0.9 Scheduler Hardening Plan

**Goal:** 在 `0.0.8` 已完成 wait/wakeup 最小平台化后，把等待与恢复逻辑进一步收敛到 runtime 内部，并为后续 timeout/fairness/多 suspension 调度做好统一入口。

**当前状态:**
- `0.0.8` 已完整交付，VM 已具备 `WaitSuspension(ctx)` 与 CLI 自动 wait/resume
- 当前自动恢复主循环仍散落在 CLI 层
- 对未实现 `Waiter` 的宿主，等待路径还缺少 runtime 级兜底策略

## 1. 版本目标

`0.0.9` 先完成三件事：

1. **统一运行入口**
   - VM 提供 `RunUntilSettled(ctx)` 统一收敛 pending/resume 主循环

2. **Wait fallback 策略**
   - 对无 `Waiter` 宿主，runtime 退回到可配置轮询等待

3. **后续调度骨架**
   - 为 timeout、fairness、多 suspension 调度预留配置位与结构边界

## 2. Milestones

### Milestone 1：RunUntilSettled 统一入口

**状态:** 进行中。

**目标:**
- runtime 提供统一 settled 入口
- CLI 切换到 runtime API
- 保持现有 async 行为不回退

### Milestone 2：无 Waiter 宿主兜底

**状态:** 进行中。

**目标:**
- `WaitSuspension()` 在无 `Waiter` 时回退到 `Host.Poll`
- 引入最小等待间隔配置
- 增加 runtime / CLI 回归

### Milestone 3：文档与交付收口

**状态:** 待开始。

**目标:**
- 更新调度语义文档
- 评估 `0.0.9` 完成度与边界

## 3. 发布前验证

```bash
go test ./iacommon/... ./iavm/... ./ialang/...
```

## 4. 非目标范围

- 多 VM / 多 suspension 的公平调度器
- 真实 OS 事件循环
- 优先级与背压策略
