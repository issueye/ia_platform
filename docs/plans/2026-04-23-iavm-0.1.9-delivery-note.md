# IAVM 0.1.9 Delivery Note

**Version:** `0.1.9`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.9` 聚焦“HTTP status retryable mapping”，在 `0.1.8` transient network retryable mapping 基础上完成了以下目标：

- `network.http_fetch` 已支持 capability 级 `retry_http_statuses`
- 命中的 HTTP 状态码会被 `DefaultHost` 转成 retryable error
- 未显式配置时，HTTP 状态仍保持原始响应语义
- runtime allowlisted `host.call` 已可直接消费这一映射

## 2. 主要交付物

### 2.1 代码

- `DefaultHost` capability-level HTTP status retry mapping
- `retry_http_statuses` / `retryHTTPStatuses` 解析
- runtime `503 -> retry -> 200` 闭环回归

### 2.2 测试

- host API HTTP status retryable 回归
- 未配置时保持原始 HTTP 响应回归
- runtime + DefaultHost 集成回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

## 4. 已知边界

以下内容仍不属于 `0.1.9` 的完整目标范围，应进入后续版本：

- 默认对所有 4xx/5xx 自动重试
- 更复杂的 HTTP retry policy
- retry 与 circuit breaker / budget policy 的联动

## 5. 版本结论

`0.1.9` 已让 IAVM 的 retryable error 机制从“网络错误类别”进一步推进到“显式 HTTP 响应状态”，同时保持默认 `http_fetch` 语义兼容。
