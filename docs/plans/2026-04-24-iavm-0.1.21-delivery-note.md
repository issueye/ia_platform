# IAVM 0.1.21 Delivery Note

**Version:** `0.1.21`  
**Date:** `2026-04-24`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.21` 聚焦“capability retry disable”，为单个 capability 补上了退出 inherited retry/backoff 策略的总开关：

- capability 现已支持 `retry_enabled`
- `retry_enabled = false` 时，该 capability 的 `poll/wait` 与 allowlisted `host.call` 都不再自动重试
- 其他 capability 的 retry 行为保持不变

## 2. 主要交付物

### 2.1 代码

- capability retry disable 配置解析
- capability 级 retry profile 清零逻辑
- host.call 与 poll retry disable 回归测试

### 2.2 测试

- capability host.call retry disable 回归
- capability poll retry disable 回归

## 4. 已知边界

以下内容仍不属于 `0.1.21` 的完整目标范围，应进入后续版本：

- 更细粒度的 `retry_poll_enabled` / `retry_call_enabled` 独立开关
- capability 之间的共享 retry budget 协调
- 与 circuit breaker / concurrency policy 的联动

## 5. 版本结论

`0.1.21` 让 IAVM 的 retry policy 从“只能通过参数调零规避”推进到“支持 capability 级显式关闭”，更适合混合稳定/不稳定宿主能力并存的场景。
