# IAVM 0.1.14 Delivery Note

**Version:** `0.1.14`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.14` 聚焦“HTTP status exclusion”，在 `0.1.13` default safe HTTP methods 基础上完成了以下目标：

- `network.http_fetch` 已支持 capability 级 `retry_http_excluded_statuses`
- capability 现在可以在状态码/状态族重试策略中显式挖掉具体状态
- 被排除状态会保持原始 HTTP 响应语义
- runtime allowlisted `host.call` 已可直接消费这一例外规则

## 2. 主要交付物

### 2.1 代码

- `retry_http_excluded_statuses` / `retryHTTPExcludedStatuses` 解析
- `DefaultHost` exclusion 优先级逻辑
- runtime + DefaultHost 的 exclusion 集成回归

### 2.2 测试

- host API exclusion 回归
- runtime exclusion 不重试回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

### 2.3 文档

- `docs/iavm/abi-capabilities.md`
- `docs/plans/2026-04-23-iavm-0.1.14-http-status-exclusion-plan.md`

## 4. 已知边界

以下内容仍不属于 `0.1.14` 的完整目标范围，应进入后续版本：

- 状态族级 exclusion 配置
- 更复杂的优先级合并策略
- retry 与 circuit breaker / budget policy 的联动

## 5. 版本结论

`0.1.14` 已让 IAVM 的 HTTP 重试策略从“只能追加命中条件”推进到“也能声明显式例外”，让状态族和默认策略在真实场景里更可控。
