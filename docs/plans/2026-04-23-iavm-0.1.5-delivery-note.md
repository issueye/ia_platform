# IAVM 0.1.5 Delivery Note

**Version:** `0.1.5`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.5` 聚焦“jitter policy”，在 `0.1.4` backoff curve profile 基础上完成了以下目标：

- `runtime.Options.RetryJitter` 已进入 runtime 配置面
- capability config 已可通过 `retry_jitter` 覆盖默认 jitter 策略
- pending promise 会保留触发时的 jitter profile，并在 wait / resume 路径继续生效
- 默认 `RetryJitter == 0` 时，现有 retry/backoff 行为保持不变

## 2. 主要交付物

### 2.1 代码

- `runtime.Options.RetryJitter`
- jitter backoff 计算 helper
- capability jitter profile 读取与 promise 透传
- wait / resume / fallback polling 路径统一使用 jitter profile

### 2.2 测试

- jitter 可重复性与范围回归
- capability jitter profile 持久化与恢复回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

### 2.3 文档

- `docs/iavm/abi-capabilities.md`
- `docs/plans/2026-04-23-iavm-0.1.5-jitter-policy-plan.md`
- 本交付说明

## 4. 已知边界

以下内容仍不属于 `0.1.5` 的完整目标范围，应进入后续版本：

- 更复杂的 jitter 分布算法
- jitter 与 circuit breaker / budget policy 的联动
- 对 `host.call` 引入自动重试

## 5. 版本结论

`0.1.5` 已让 IAVM 的 retry/backoff 从“可配置退避曲线”推进到“可配置退避曲线 + 有界抖动”，同时保持默认行为稳定兼容。
