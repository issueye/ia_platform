# IAVM 0.1.8 Delivery Note

**Version:** `0.1.8`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.8` 聚焦“transient network retryable mapping”，在 `0.1.7` retryable host error 基础上完成了以下目标：

- `DefaultHost` 已能对瞬时网络 `net.Error` 自动打上 retryable 标记
- 默认宿主的 network I/O 路径已可直接触发 runtime retry policy
- policy / 参数 / capability 类错误仍保持非 retryable
- runtime 侧语义无需再额外调整，直接复用现有 classifier

## 2. 主要交付物

### 2.1 代码

- `DefaultHost` transient network error marker
- network provider / handle 错误自动映射
- network transient error 回归测试

### 2.2 测试

- `http_fetch/dial/accept/send/recv` 自动打标回归
- policy 错误非 retryable 回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

### 2.3 文档

- `docs/iavm/abi-capabilities.md`
- `docs/plans/2026-04-23-iavm-0.1.8-transient-network-retry-plan.md`
- 本交付说明

## 3. 发布前验证

`0.1.8` release gate:

```bash
go test ./iacommon/pkg/host/api

```

## 4. 已知边界

以下内容仍不属于 `0.1.8` 的完整目标范围，应进入后续版本：

- 对 FS 错误做自动 retryable 映射
- 更细粒度的网络 error taxonomy
- retry 与 circuit breaker / budget policy 的联动

## 5. 版本结论

`0.1.8` 已让 IAVM 的 retryable error 机制从“runtime 可识别”推进到“DefaultHost 默认网络路径可直接产出”，使前几版重试策略在默认宿主上真正闭环。
