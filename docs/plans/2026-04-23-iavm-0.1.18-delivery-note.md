# IAVM 0.1.18 Delivery Note

**Version:** `0.1.18`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.18` 聚焦“host.call retry exclusion”，为 runtime 的 operation 级重试策略补上了 exclusion 能力：

- runtime 现已支持 `Options.RetryExcludedCallOps`
- capability 现已支持 `retry_excluded_call_ops`
- 命中 exclusion 的 `host.call` operation 不再触发自动重试

## 2. 主要交付物

### 2.1 代码

- runtime host.call exclusion 配置解析
- allowlist/exclusion 优先级逻辑
- runtime host.call exclusion 回归测试

### 2.2 测试

- 默认 allowlist + exclusion 冲突回归
- capability allowlist + exclusion 冲突回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

### 2.3 文档

- `docs/iavm/abi-capabilities.md`
- `docs/plans/2026-04-23-iavm-0.1.18-host-call-retry-exclusion-plan.md`
- 本交付说明

## 3. 发布前验证

`0.1.18` release gate:

```bash
go test ./iacommon/pkg/host/api ./iavm/pkg/runtime
go test ./iacommon/... ./iavm/... ./ialang/...
```

## 4. 已知边界

以下内容仍不属于 `0.1.18` 的完整目标范围，应进入后续版本：

- 更复杂的 host.call retry policy DSL
- 基于 operation 语义自动推断 exclusion
- 与 circuit breaker / concurrency policy 的联动

## 5. 版本结论

`0.1.18` 让 IAVM 的 host.call retry 从“只有 allowlist”推进到“allowlist + exclusion”组合控制，能更细粒度地约束自动重试范围。
