# IAVM 0.1.24 Capability Wait Retry Disable Plan

**Goal:** 为 capability 增加 `retry_wait_enabled` 细粒度开关，让单个 capability 可以只关闭 `wait(handle)` 自动重试，而保留 `poll` retry。

**当前状态:**
- `0.1.23` 已完整交付，capability 已支持 `retry_poll_enabled`
- 当前若只想关闭 `wait(handle)` retry、同时保留 `poll` retry，仍只能一起关闭

## 1. 版本目标

`0.1.24` 完成三件事：

1. **Capability wait-retry disable 配置**
   - 支持 `Config.retry_wait_enabled`
   - 支持 `Config.retryWaitEnabled`
   - 为 `false` 时只关闭该 capability 的 `wait(handle)` 自动重试

2. **Wait / poll 闭环**
   - `wait(handle)` 在 capability 级 wait-retry disable 下回退为单次尝试
   - 同一 capability 的 `poll` 仍可继续使用 retry/backoff profile
   - 若仅设置 `retry_poll_enabled` 而未设置 `retry_wait_enabled`，则继续保持旧版本的联动语义

3. **测试与文档**
   - 增加 runtime capability wait-retry disable 回归
   - 更新 ABI / 计划文档 / delivery note

## 2. Milestones

### Milestone 1：Capability wait-retry disable

**状态:** 已完成。

**已完成内容:**
- capability profile 与 pending promise state 已支持 `retry_wait_enabled`
- `wait(handle)` 与 `poll` 的 retry 开关已拆分
- 保留了 `retry_poll_enabled` 对 `wait` 的历史兼容行为

### Milestone 2：运行时闭环

**状态:** 已完成。

**已完成内容:**
- capability wait retry disable 回归已补齐
- 显式开启 wait retry 覆盖 poll-disable 联动的回归已补齐
- `poll` retry 保持生效的回归已补齐

### Milestone 3：文档收口

**状态:** 已完成。

**已完成内容:**
- ABI 文档已补充 capability wait-retry disable 开关
- 计划文档与 delivery note 已补齐
