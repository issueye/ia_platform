# IAVM 0.1.0 Delivery Note

**Version:** `0.1.0`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.0` 聚焦“timeout / cancellation 基线”，在 `0.0.9` 调度主循环统一收敛基础上完成了以下目标：

- runtime 已把 host acquire/call/poll 统一绑定到同一执行 context
- `RunUntilSettled(ctx)` 已尊重外部 timeout / cancellation
- `Options.MaxDuration` 已形成 runtime 级 deadline 兜底
- runtime / CLI 都已具备 `context deadline exceeded` 级别的回归保护

## 2. 主要交付物

### 2.1 代码

- runtime 内部统一执行上下文
- settled wait/resume 链路的 timeout / cancel 透传
- `MaxDuration` 对整个 settled 执行过程生效

### 2.2 测试

- runtime 外部 timeout 回归
- runtime `MaxDuration` 回归
- CLI cancellation 回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

### 2.3 文档

- `docs/iavm/abi-capabilities.md`
- `docs/plans/2026-04-23-iavm-0.1.0-timeout-cancellation-plan.md`
- 本交付说明

## 3. 发布前验证

`0.1.0` release gate:

```bash
go test ./iacommon/... ./iavm/... ./ialang/...
```

## 4. 已知边界

以下内容仍不属于 `0.1.0` 的完整目标范围，应进入后续版本：

- 抢占式取消
- 更细粒度的 per-operation timeout policy
- 多任务公平调度与优先级策略

## 5. 版本结论

`0.1.0` 已让 IAVM 拥有首个真正可验证的 timeout / cancellation 执行基线，后续可以在这个基础上继续扩展更强的 scheduler 与 policy 能力。
