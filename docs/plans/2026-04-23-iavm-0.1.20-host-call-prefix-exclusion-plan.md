# IAVM 0.1.20 Host Call Prefix Exclusion Plan

**Goal:** 为 runtime 的 `host.call` retry policy 增加前缀 exclusion，让 operation family 级别的排除规则可以与 exact/prefix allowlist 组合使用。

**当前状态:**
- `0.1.19` 已完整交付，`host.call` retry 已支持 exact allowlist、prefix allowlist 与 exact exclusion
- 当前若想屏蔽整个 `fs.`、`network.` family 的自动重试，仍需要逐条枚举 exact exclusion

## 1. 版本目标

`0.1.20` 完成三件事：

1. **Host call prefix exclusion 配置**
   - 支持 `Options.RetryExcludedCallOpPrefixes`
   - 支持 capability `retry_excluded_call_op_prefixes`
   - 允许通过前缀批量排除 operation family

2. **Runtime 闭环**
   - prefix exclusion 可覆盖 exact allowlist 与 prefix allowlist
   - 命中排除前缀的 operation 保持单次尝试

3. **测试与文档**
   - 增加 runtime prefix exclusion precedence 回归
   - 更新 ABI / 计划文档 / delivery note

## 2. Milestones

### Milestone 1：Runtime prefix exclusion

**状态:** 已完成。

**已完成内容:**
- runtime options 与 capability profile 已支持 excluded call op prefixes
- prefix exclusion 在 allow 命中后统一生效
- 命中排除前缀时不再进入 host.call retry

### Milestone 2：优先级回归

**状态:** 已完成。

**已完成内容:**
- exact allowlist + prefix exclusion 冲突回归已补齐
- capability prefix allowlist + prefix exclusion 冲突回归已补齐
- prefix exclusion 继续高优先级覆盖 allow 规则

### Milestone 3：文档收口

**状态:** 已完成。

**已完成内容:**
- ABI 文档已补充 prefix exclusion 能力
- 计划文档与 delivery note 已补齐
