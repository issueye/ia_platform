# IAVM 0.1.3 Delivery Note

**Version:** `0.1.3`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.3` 聚焦“retry / backoff 最小策略”，在 `0.1.2` capability timeout profile 基础上完成了以下目标：

- `RetryCount` / `RetryBackoff` 已进入 runtime Options
- capability config 已可覆盖默认 retry/backoff
- retry/backoff 只对可安全重试的 `poll/wait` 超时路径生效
- pending promise 会保留触发时的 retry/backoff profile，并在恢复路径中继续使用

## 2. 主要交付物

### 2.1 代码

- `runtime.Options.RetryCount`
- `runtime.Options.RetryBackoff`
- poll/wait 重试与 backoff helper
- capability retry profile 读取与 promise 透传

### 2.2 测试

- poll timeout retry 回归
- wait timeout retry 回归
- capability retry profile 覆盖回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

## 4. 已知边界

以下内容仍不属于 `0.1.3` 的完整目标范围，应进入后续版本：

- `host.call` 自动重试
- 更复杂的 jitter/backoff 算法
- retry 与 circuit breaker / policy engine 的组合策略

## 5. 版本结论

`0.1.3` 已在 timeout policy 之上，为 IAVM 增加了最小可控的 retry/backoff 机制，并保持了对副作用操作的保守边界。
