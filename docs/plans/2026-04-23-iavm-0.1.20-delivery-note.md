# IAVM 0.1.20 Delivery Note

**Version:** `0.1.20`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.20` 聚焦“host.call prefix exclusion”，为 runtime 的 operation 级重试策略补上了前缀排除能力：

- runtime 现已支持 `Options.RetryExcludedCallOpPrefixes`
- capability 现已支持 `retry_excluded_call_op_prefixes`
- 命中排除前缀的 `host.call` operation 不再触发自动重试

## 2. 主要交付物

### 2.1 代码

- runtime host.call prefix exclusion 配置解析
- exact/prefix allow + exact/prefix exclusion 优先级逻辑
- runtime prefix exclusion 回归测试

### 2.2 测试

- exact allowlist + prefix exclusion 冲突回归
- capability prefix allowlist + prefix exclusion 冲突回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

### 2.3 文档

- `docs/iavm/abi-capabilities.md`
- `docs/plans/2026-04-23-iavm-0.1.20-host-call-prefix-exclusion-plan.md`
- 本交付说明

## 3. 发布前验证

`0.1.20` release gate:

```bash
go test ./iacommon/pkg/host/api ./iavm/pkg/runtime
go test ./iacommon/... ./iavm/... ./ialang/...
```

## 4. 已知边界

以下内容仍不属于 `0.1.20` 的完整目标范围，应进入后续版本：

- 更复杂的 glob / regex exclusion matching
- allow/exclude 优先级 DSL
- 与 circuit breaker / concurrency policy 的联动

## 5. 版本结论

`0.1.20` 让 IAVM 的 host.call retry policy 在 operation family 维度同时具备 allow 与 exclude 能力，使策略收敛方式更加完整。
