# IAVM 0.1.12 Delivery Note

**Version:** `0.1.12`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.12` 聚焦“HTTP status class retry policy”，在 `0.1.11` HTTP retry method guard 基础上完成了以下目标：

- `network.http_fetch` 已支持 capability 级 `retry_http_status_classes`
- capability 现在可以更简洁地声明 `5xx` 这类状态族重试策略
- status class、精确 status list 和 method guard 已可组合使用
- runtime allowlisted `host.call` 已可直接消费该状态族配置

## 2. 主要交付物

### 2.1 代码

- `retry_http_status_classes` / `retryHTTPStatusClasses` 解析
- `DefaultHost` status class 判定逻辑
- runtime + DefaultHost 的 status class 集成回归

### 2.2 测试

- host API status class 命中回归
- host API non-match 保持原始响应回归
- runtime `5xx -> retry -> 200` 集成回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

- `docs/plans/2026-04-23-iavm-0.1.12-http-status-class-plan.md`

## 4. 已知边界

以下内容仍不属于 `0.1.12` 的完整目标范围，应进入后续版本：

- 更复杂的状态码范围语法
- 默认自动把所有 `5xx` 视为可重试
- retry 与 circuit breaker / budget policy 的联动

## 5. 版本结论

`0.1.12` 已让 IAVM 的 HTTP 重试策略从“枚举单个状态码”推进到“声明状态族”，提高了 capability 配置的表达力，同时保持原有安全边界。
