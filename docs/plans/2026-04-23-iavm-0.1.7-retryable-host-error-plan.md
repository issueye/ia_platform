# IAVM 0.1.7 Retryable Host Error Plan

**Goal:** 在 `0.1.6` 已完成 safe `host.call` retry allowlist 之后，为 IAVM 增加“宿主显式可重试错误”支持，让 retry 判定不再只依赖 `context deadline exceeded`。

**当前状态:**
- `0.1.6` 已完整交付，`poll/wait` 与 allowlisted `host.call` 已支持 timeout/backoff/jitter profile
- 当前 retry 判定仍偏窄，主要只认 timeout 类错误

## 1. 版本目标

`0.1.7` 完成三件事：

1. **Host retryable error marker**
   - 提供 `api.MarkRetryable(err)`
   - 提供 `api.IsRetryableError(err)`

2. **Runtime retry classifier**
   - `poll/wait` 支持 retryable host error
   - allowlisted `host.call` 支持 retryable host error

3. **测试与文档**
   - 增加 retryable host error 回归
   - 更新 ABI / 计划文档 / delivery note

## 2. Milestones

### Milestone 1：Host API 标记层

**状态:** 已完成。

**已完成内容:**
- `iacommon/pkg/host/api` 已提供 retryable error marker
- marker 保留底层错误链，便于 `errors.Is/As` 继续工作
- marker 支持幂等包装

### Milestone 2：Runtime retry classifier

**状态:** 已完成。

**已完成内容:**
- runtime retry 判定已从“只认 deadline exceeded”扩展为“deadline exceeded 或宿主显式 retryable”
- `poll/wait` 已接入新判定逻辑
- allowlisted `host.call` 已接入新判定逻辑

### Milestone 3：测试与文档收口

**状态:** 已完成。

**已完成内容:**
- host API marker 回归已补齐
- `poll/wait` / `host.call` retryable error 回归已补齐
- runtime 与全量测试已通过

## 3. 发布前验证
