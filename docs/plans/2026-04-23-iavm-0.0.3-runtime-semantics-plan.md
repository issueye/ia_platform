# IAVM 0.0.3 Runtime Semantics Completion Plan

**Goal:** 在 `0.0.2` 平台硬化基础上，推进 `0.0.3` 解决 IAVM runtime 的核心语义缺陷，使更多 ialang 示例能稳定通过 build/verify/run 全流程。重点修复值运算、类型强转、闭包捕获和错误处理四大核心问题。

**核心交付物:**
- 完整的跨类型值运算（string + number、I64 + F64、F64 数组索引等）
- 基本的闭包 / upvalue 捕获支持
- try/catch 的 catch 变量绑定
- 至少 2 个旧示例从"不可运行"变为"可运行"
- 更大的示例矩阵与回归测试覆盖

**原则:** 优先修复已有 opcodes 的 runtime 行为，不引入新 opcodes；所有修复都必须有回归测试和示例验证。

---

## 1. 0.0.2 之后的基线与遗留问题

### 1.1 已完成基线

`0.0.2` 已形成以下稳定基础：

- verifier/profile 语义统一到共享 `BuildVerifyOptions` 路径
- `verify-iavm` / `inspect-iavm --verify` 输出包含 policy 摘要
- capability allowlist 语义明确（allow-all / deny-all / allowlist）
- 4 个稳定 IAVM 示例进入 smoke matrix
- CLI 错误分层前缀（`[compile]` / `[decode]` / `[verify]` / `[runtime]`）
- `hello.ia` 兼容性问题已定性（外部模块导入不支持）

### 1.2 当前主要遗留问题

`0.0.3` 需要优先处理以下 P0 级语义缺陷：

1. **string + number 拼接返回 null**
   - `addValues` 不处理 `ValueString` 与任何其他类型的 `+` 运算
   - 影响：所有使用 `print("result: " + x)` 模式的示例
   - 位置：`iavm/pkg/runtime/interpreter.go:538-549`

2. **I64 + F64 混合算术返回 null**
   - `addValues` / `subValues` / `mulValues` / `divValues` 要求左右类型严格匹配
   - 影响：循环计数器（I64）与字面量（F64）的运算失败
   - 位置：`iavm/pkg/runtime/interpreter.go:538-595`

3. **I64 vs F64 比较运算静默失败**
   - `valuesEqual` / `valuesLess` / `valuesGreater` 等不处理跨类型
   - 影响：`let i = 0; while (i < 5)` 当 `i` 为 I64、`5` 为 F64 时条件始终 false
   - 位置：`iavm/pkg/runtime/interpreter.go:623-658`

4. **F64 数组/字符串索引被拒绝**
   - `OpIndex` 仅接受 `ValueI64`，不接受 `ValueF64`
   - 影响：`arr[0]`（`0` 编译为 `float64`）失败
   - 位置：`iavm/pkg/runtime/interpreter.go:454,476`

5. **闭包不捕获 upvalue**
   - `OpClosure` 仅压入 FuncRef，无词法环境捕获
   - 影响：`closure.ia` 完全无法工作
   - 位置：`iavm/pkg/runtime/interpreter.go:190-191`

6. **函数体内 OpClosure 降级为 OpNop**
   - `lowerFunction` 传入 `funcMap=nil`，内部闭包指令被丢弃
   - 影响：函数内定义的嵌套函数无法被引用
   - 位置：`iavm/pkg/bridge/ialang/compiler_lowering.go:553-563`

7. **catch 变量未绑定**
   - `OpPushTry` 仅存储 handler IP，忽略 catch 变量名
   - 影响：`catch (err) { print(err) }` 中 `err` 无法访问
   - 位置：`iavm/pkg/runtime/interpreter.go:441-442`

## 2. 0.0.3 目标范围

### 2.1 必须完成

1. **值运算跨类型支持**
   - `addValues` 支持 string + string、string + number 隐式转换
   - 所有算术运算支持 I64 + F64 自动提升
   - 所有比较运算支持 I64 / F64 交叉比较

2. **数组/字符串索引 F64 兼容**
   - `OpIndex` 接受 F64 索引，自动截断为整数

3. **基本闭包支持**
   - `OpClosure` 捕获当前帧的词法环境（至少支持全局变量访问）
   - `lowerFunction` 传递 `funcMap` 以支持函数体内的闭包指令

4. **catch 变量绑定**
   - 异常恢复时将异常值绑定到 catch 变量对应的 local slot

5. **至少 2 个旧示例进入 IAVM 矩阵**
   - 优先目标：`control.ia`（完整输出）、`try_catch.ia`（基本 try/catch 部分）

6. **所有修复配套回归测试**
   - 每个修复项至少 2 个单元测试
   - 每个新进入矩阵的示例有 CLI 回归测试

### 2.2 非目标范围

以下内容不进入 `0.0.3`：

- 完整 class / inheritance / `this` 绑定
- 完整 async / await / Promise 支持
- 外部模块导入解析系统
- finally 块执行保证
- spread 语法支持
- JIT / AOT / native backend
- 生产级安全沙箱

## 3. 0.0.3 交付路径

### Milestone 1：值运算跨类型支持

**目标:** 让 IAVM runtime 的算术和比较运算支持 I64/F64 混合和 string 拼接。

**任务:**

1. 修改 `addValues`（`interpreter.go:538-549`）：
   - 添加 `ValueString` + `ValueString` → string 拼接
   - 添加 `ValueString` + 任意 → string 拼接（通过 `valueToString`）
   - 添加 `ValueI64` + `ValueF64` → F64 提升
   - 添加 `ValueF64` + `ValueI64` → F64 提升

2. 修改 `subValues` / `mulValues` / `divValues` / `modValues`：
   - I64 + F64 → 双方转 F64 后运算

3. 修改 `valuesEqual` / `valuesLess` / `valuesGreater` / `valuesLessEqual` / `valuesGreaterEqual`：
   - I64 / F64 交叉比较：统一转 F64 比较

4. 修改 `OpIndex`（`interpreter.go:449-488`）：
   - 接受 `ValueF64` 索引，截断为 `int64` 使用

**验收标准:**
- `"hello " + "world"` → `"hello world"`
- `"count: " + 42` → `"count: 42"`
- `int64(3) + float64(2.5)` → `float64(5.5)`
- `int64(3) < float64(5)` → `true`
- `arr[0.0]` 正常工作
- 新增测试覆盖以上场景

### Milestone 2：闭包支持

**目标:** 让 IAVM 支持基本的闭包——至少能捕获全局变量，嵌套函数能被引用和调用。

**任务:**

1. 修改 `lowerFunction`（`compiler_lowering.go:189-274`）：
   - 传递 `funcMap` 而非 `nil`，使函数体内的 `OpClosure` 能正确降级
   - 需要确保 `funcIndexMap` 在 `LowerToModule` 中对 `lowerFunction` 可用

2. 增强 `OpClosure` 在解释器中的行为（`interpreter.go:190-191`）：
   - 当前仅压入 FuncRef
   - Phase 1（本 milestone）：确保嵌套函数引用正确（不做 upvalue 捕获）
   - 验证：函数返回的嵌套函数可以被调用

3. （可选 Phase 2）添加 upvalue 捕获：
   - 在 `Frame` 中添加 `Upvalues` 字段
   - `OpClosure` 时捕获当前 frame 的 locals 引用
   - 对全局变量的访问通过 `OpLoadGlobal` / `OpStoreGlobal` 天然共享

**验收标准:**
- `let f = function() { return 42; }; f()` → `42`（已在 matrix 中，确保仍通过）
- `function makeAdder(x) { return function(y) { return x + y; }; } makeAdder(3)(4)` → `7`（如果 upvalue 完成）
- 至少嵌套函数引用和调用正确

### Milestone 3：catch 变量绑定与错误处理增强

**目标:** 让 try/catch 的 catch 子句能正确绑定异常变量。

**任务:**

1. 修改 `OpPushTry`（`interpreter.go:441-451`）：
   - 将 `inst.B`（catch 变量名 constant index）存入 TryHandler
   - 修改 `Frame.TryHandlers` 从 `[]uint32`（IP only）改为包含 IP + catchLocalIdx 的结构

2. 修改异常恢复逻辑（`interpreter.go:36-59`）：
   - 跳转到 handler 后，将异常值存入 catch 变量对应的 local slot

3. 添加 `try/catch` 在 IAVM 中的集成测试

**验收标准:**
- `try { throw "error"; } catch(e) { print(e); }` → 打印 `"error"`
- catch 变量在 catch 块结束后不再污染作用域
- 回归测试覆盖

### Milestone 4：示例矩阵扩展

**目标:** 将更多旧示例纳入 IAVM smoke matrix。

**候选示例:**

| 示例 | 当前状态 | 0.0.3 目标 | 依赖 Milestone |
|------|----------|-----------|---------------|
| `control.ia` | 输出部分错误（string + number = null） | 完整正确输出 | M1 |
| `try_catch.ia` | 基本结构可运行但 catch 变量不可用 | 基本 try/catch 可运行 | M3 |
| `comparison.ia` | string + number = null | 完整正确输出 | M1 |
| `operators.ia` | string + number = null | 完整正确输出 | M1 |
| `data.ia` | arr[0] 索引类型失败 | 完整正确输出 | M1 |

**任务:**
- 为每个纳入矩阵的示例运行 build/verify/run，确认输出
- 为每个新增示例添加 CLI 回归测试
- 对仍不可运行的示例记录具体原因

**验收标准:**
- 至少 2 个旧示例（非 `iavm_*` 前缀）进入稳定 smoke matrix
- smoke matrix 总数达到 6+
- 每个新增示例有 CLI 回归测试

## 5. 风险与处理策略

| 风险 | 影响 | 处理策略 |
|------|------|----------|
| 闭包 upvalue 捕获实现复杂度可能超预期 | 可能需要拆分到 Phase 2 | 先完成函数引用传递，upvalue 捕获作为可选交付 |
| 跨类型运算可能引入精度问题 | F64 转换可能丢失 I64 精度 | 遵循 JS 语义：I64 + F64 → F64，仅 string + number 隐式转 string |
| 旧示例可能有其他未发现的依赖问题 | 更多示例不可运行 | 逐个验证，不可运行的记录原因 |
| 修改 interpreter 可能影响已有测试 | 0.0.2 回归 | 先运行全量测试建立基线，修改后逐个验证 |

## 7. 0.0.3 之后的方向

当 `0.0.3` 完成后，再考虑进入更高一层的平台推进：

1. **完整闭包 upvalue 捕获**
   - 如果 0.0.3 仅完成函数引用传递，0.0.4 需完成 upvalue cell 捕获

2. **class / this 绑定**
   - `OpNew` 创建 `this` 对象
   - `OpClass` 设置原型链和方法绑定

3. **Host ABI 完善**
   - `OpHostCall` 传递实际参数
   - 多 capability 并发支持

4. **finally 块执行保证**
   - 在异常恢复路径中确保 finally 块执行

5. **更多示例兼容性修复**
   - `closure.ia`、`class.ia`、`data.ia`、`operators.ia` 全量兼容
