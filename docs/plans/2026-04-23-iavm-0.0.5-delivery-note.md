# IAVM 0.0.5 Delivery Note

**Version:** `0.0.5`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.0.5` 聚焦“平台契约收敛”，不扩张到新的大块语言特性。当前版本完成了以下目标：

- `OpHostCall` 支持从栈上传递业务参数
- 单对象参数可直接映射为 `CallRequest.Args`
- `finally` / `try-catch-finally` 关键路径具备稳定回归保护
- `DefaultHost` 已接入 FS typed adapter 与 Network HTTP typed adapter
- verifier 已校验静态 host operation 与 capability 声明的一致性
- runtime 已稳定绑定最近一次导入的 capability，并兼容模块级常量池

## 2. 主要交付物

### 2.1 代码

- Host ABI 参数传递增强
- FS / Network HTTP typed request-response adapter
- capability-operation 静态一致性校验
- capability import / host call 运行时稳定化

### 2.2 测试

- runtime capability 参数传递与 capability 选择测试
- integration encode/decode 后的 Host ABI 测试
- verifier capability 一致性测试
- CLI `verify-iavm` capability 校验回归
- CLI `try_catch.ia` finally 回归

### 2.3 文档

- `docs/iavm/abi-capabilities.md`
- `docs/plans/2026-04-23-iavm-0.0.5-host-abi-control-flow-plan.md`
- `ialang/README.md`

## 3. 发布前验证

推荐使用以下命令作为 `0.0.5` release gate：

```bash
make test-release-0.0.5
```

等价命令：

```bash
go test ./iacommon/... ./iavm/... ./ialang/...
```

## 4. 非目标范围

以下内容不属于 `0.0.5` 交付范围，应进入后续版本：

- `extends` / `super`
- Host handle table 与 `OpHostPoll` 的完整异步资源模型
- 更完整的 async/await 平台化
- FS / Network 其余 operation 的全面 typed adapter 化

## 5. 后续版本建议

`0.0.6` 建议转入“语义扩展”阶段，优先顺序：

1. `extends` / `super`
2. Host handle table / poll 模型
3. 更广的 typed adapter 与 CLI capability 配置映射
