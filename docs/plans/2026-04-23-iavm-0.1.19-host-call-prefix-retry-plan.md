# IAVM 0.1.19 Host Call Prefix Retry Plan

**Goal:** 为 runtime 的 `host.call` retry policy 增加前缀 allowlist，让同一 operation family 可以批量放行自动重试，而不必逐个枚举完整 operation 名称。

**当前状态:**
- `0.1.18` 已完整交付，`host.call` retry 已支持 exact allowlist 与 exact exclusion
- 当前若要让 `fs.*`、`network.*` 这类 family 进入 retry allowlist，仍需要逐个 operation 枚举

## 1. 版本目标

`0.1.19` 完成三件事：

1. **Host call prefix allowlist 配置**
   - 支持 `Options.RetryCallOpPrefixes`
   - 支持 capability `retry_call_op_prefixes`
   - 允许通过前缀批量声明可重试 operation family

2. **Runtime 闭环**
   - exact allowlist 与 prefix allowlist 二选一命中即可放行 retry
   - exact exclusion 继续高优先级覆盖 prefix allowlist

3. **测试与文档**
   - 增加 runtime prefix allowlist 与 exclusion precedence 回归
   - 更新 ABI / 计划文档 / delivery note

## 2. Milestones

### Milestone 1：Runtime prefix allowlist

**状态:** 已完成。

**已完成内容:**
- runtime options 与 capability profile 已支持 call op prefix allowlist
- prefix allowlist 通过 `strings.HasPrefix` 匹配 operation family
- 未命中 exact allowlist 但命中 prefix allowlist 时同样允许 retry

### Milestone 2：优先级回归

**状态:** 已完成。

**已完成内容:**
- 默认 prefix allowlist retry 回归已补齐
- capability prefix allowlist + exact exclusion 冲突回归已补齐
- exclusion 仍优先于 prefix 放行

### Milestone 3：文档收口

**状态:** 已完成。

**已完成内容:**
- ABI 文档已补充 prefix allowlist 能力
- 计划文档与 delivery note 已补齐
