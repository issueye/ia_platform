# IAVM 0.1.1 Delivery Note

**Version:** `0.1.1`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.1` 聚焦“operation timeout policy”，在 `0.1.0` timeout / cancellation 基线之上完成了以下目标：

- `HostTimeout` 已作用于单次 acquire/call/poll
- `WaitTimeout` 已作用于单次 `Wait(handle)`
- 恢复路径中的 poll/wait 已纳入操作级超时保护
- runtime 已明确 operation timeout 与总 deadline 的组合边界

## 2. 主要交付物

### 2.1 代码

- `runtime.Options.HostTimeout`
- `runtime.Options.WaitTimeout`
- host operation 上下文封装与恢复路径超时透传

### 2.2 测试

- blocked `host.call` 超时回归
- blocked `host.poll` 超时回归
- blocked `wait(handle)` 超时回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

### 2.3 文档

- `docs/iavm/abi-capabilities.md`
- `docs/plans/2026-04-23-iavm-0.1.1-operation-timeout-plan.md`
- 本交付说明

## 3. 发布前验证

`0.1.1` release gate:

```bash
go test ./iacommon/... ./iavm/... ./ialang/...
```

## 4. 已知边界

以下内容仍不属于 `0.1.1` 的完整目标范围，应进入后续版本：

- per-capability 动态 timeout profile
- 优先级感知的 timeout 策略
- retry/backoff 与更高级的故障恢复机制

## 5. 版本结论

`0.1.1` 已在总 deadline 之外，为 IAVM 增加了单次 host operation 级别的超时保护，让 runtime 在慢调用和卡住等待面前更可控。
