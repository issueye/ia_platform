# IAVM 0.1.10 Retry-After Hint Plan

**Goal:** 在 `0.1.9` 已完成 HTTP 状态码 retryable 映射之后，为 retryable error 增加 backoff hint，让 `Retry-After` 这类宿主提示能直接影响 runtime 的退避时间。

**当前状态:**
- `0.1.9` 已完整交付，`network.http_fetch` 已支持显式 HTTP 状态码映射为 retryable error
- runtime 当前虽然能识别 retryable error，但仍只会使用本地配置的 backoff/jitter 公式

## 1. 版本目标

`0.1.10` 完成三件事：

1. **Retryable backoff hint**
   - 支持 `api.MarkRetryableAfter(err, backoff)`
   - 支持 `api.RetryBackoffHint(err)`

2. **Runtime backoff 消费**
   - 有 hint 时优先使用宿主提示
   - 无 hint 时继续走默认 backoff/jitter

3. **HTTP Retry-After 接入**
   - `network.http_fetch` 在命中 retryable status 且存在 `Retry-After` 时透传 hint
   - 增加 host API 与 runtime 回归

## 2. Milestones

### Milestone 1：Retryable hint 标记层

**状态:** 已完成。

**已完成内容:**
- host API 已支持 retryable backoff hint
- hint 与 retryable 标记可同时保留底层错误链
- 无 hint 场景保持旧行为不变

### Milestone 2：Runtime backoff 消费

**状态:** 已完成。

**已完成内容:**
- runtime 已支持优先消费宿主 backoff hint
- hint 仍受 max backoff cap 约束
- hint 命中时不会再叠加本地 jitter

### Milestone 3：HTTP Retry-After 收口

**状态:** 已完成。

**已完成内容:**
- `DefaultHost` 已支持从 `Retry-After` 提取 backoff hint
- host API 与 runtime 回归已补齐
- 全量测试已通过

## 3. 发布前验证

```bash
go test ./iacommon/pkg/host/api ./iavm/pkg/runtime
go test ./iacommon/... ./iavm/... ./ialang/...
```

## 4. 非目标范围

- 更复杂的 backoff hint 合并策略
- 默认对所有 HTTP 状态自动消费 Retry-After
- retry 与 circuit breaker / budget policy 的联动
