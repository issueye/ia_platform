# IAVM 0.1.3 Retry Backoff Plan

**Goal:** 在 `0.1.2` 已建立 capability timeout profile 后，为 IAVM 增加最小 retry/backoff 策略，但只覆盖可安全重试的 `poll/wait` 类操作。

**当前状态:**
- `0.1.2` 已完整交付，capability timeout profile 已稳定
- 当前 runtime 在 poll/wait 超时后会直接失败，缺少最小自动恢复策略

## 1. 版本目标

`0.1.3` 先完成三件事：

1. **全局 retry/backoff**
   - `RetryCount` / `RetryBackoff` 进入 runtime Options

2. **Capability retry profile**
   - capability config 可覆盖默认 retry/backoff

3. **安全重试边界**
   - 仅对 `poll/wait` 的超时错误重试
   - 不对可能有副作用的 `host.call` 自动重试

## 2. Milestones

### Milestone 1：Poll retry/backoff

**状态:** 进行中。

**目标:**
- poll 超时可按策略重试
- 恢复路径中的 poll 也复用同一策略

### Milestone 2：Wait retry/backoff

**状态:** 进行中。

**目标:**
- wait 超时可按策略重试
- capability profile 可覆盖默认 retry/backoff

### Milestone 3：文档与交付收口

**状态:** 待开始。

**目标:**
- 更新 ABI / delivery 文档
- 评估 `0.1.3` 完成度与边界

## 3. 发布前验证

```bash
go test ./iacommon/... ./iavm/... ./ialang/...
```

## 4. 非目标范围

- `host.call` 自动重试
- 更复杂的 jitter/backoff 算法
- retry 与 circuit breaker 组合策略
