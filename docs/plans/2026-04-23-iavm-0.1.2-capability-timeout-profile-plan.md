# IAVM 0.1.2 Capability Timeout Profile Plan

**Goal:** 在 `0.1.1` 已建立 operation timeout policy 后，为 IAVM 增加 capability 级别的 timeout profile，让不同 capability 可以覆盖默认的 host / wait 超时。

**当前状态:**
- `0.1.1` 已完整交付，runtime 已具备全局 `HostTimeout` / `WaitTimeout`
- 当前 timeout policy 仍偏全局，缺少 capability 自身配置的差异化控制

## 1. 版本目标

`0.1.2` 先完成三件事：

1. **Capability timeout profile**
   - 从 capability `Config` 读取 `host_timeout_ms` / `wait_timeout_ms`

2. **Promise 挂起透传**
   - pending promise 记住触发时的 timeout profile

3. **测试与文档**
   - 增加 capability profile 覆盖回归
   - 更新 ABI / 计划文档

## 2. Milestones

### Milestone 1：Capability host timeout

**状态:** 已完成。

**已完成内容:**
- capability config 已可覆盖默认 `HostTimeout`
- `host.call` / `host.poll` / acquire 路径已生效
- blocked host.call 的 capability profile 覆盖回归已完成

### Milestone 2：Capability wait timeout

**状态:** 已完成。

**已完成内容:**
- capability config 已可覆盖默认 `WaitTimeout`
- pending promise 恢复路径会沿用触发时的 profile
- blocked wait 的 capability profile 覆盖回归已完成

### Milestone 3：文档与交付收口

**状态:** 已完成。

**已完成内容:**
- ABI / profile 语义文档已更新
- `0.1.2` delivery note 已生成
- 当前版本完成度与边界已明确

## 3. 发布前验证

```bash
go test ./iacommon/... ./iavm/... ./ialang/...
```

## 4. 非目标范围

- 更复杂的 profile 继承机制
- profile 与 retry/backoff 的组合策略
