# IAVM 0.1.21 Capability Retry Disable Plan

**Goal:** 为 capability 增加显式 `retry_enabled` 开关，让单个 capability 可以退出默认或全局配置的 retry/backoff 策略。

**当前状态:**
- `0.1.20` 已完整交付，runtime 已支持较完整的 `host.call` retry allow/exclude 组合能力
- 当前若某个 capability 不适合重试，只能把该 capability 的每个 retry 参数逐项调成零，缺少明确的总开关

## 1. 版本目标

`0.1.21` 完成三件事：

1. **Capability retry disable 配置**
   - 支持 `Config.retry_enabled`
   - 支持 `Config.retryEnabled`
   - 为 `false` 时关闭该 capability 的 retry/backoff profile

2. **Poll/wait + host.call 闭环**
   - `poll/wait` 在 capability 级 retry disable 下回退为单次尝试
   - allowlisted `host.call` 在 capability 级 retry disable 下同样不再自动重试

3. **测试与文档**
   - 增加 runtime capability retry disable 回归
   - 更新 ABI / 计划文档 / delivery note

## 2. Milestones

### Milestone 1：Capability retry disable

**状态:** 已完成。

**已完成内容:**
- capability profile 已支持 `retry_enabled`
- 关闭时会清空当前 capability 的 retry 次数、backoff 与 host.call retry policy
- 不影响其他 capability 的默认 retry 行为

### Milestone 2：运行时闭环

**状态:** 已完成。

**已完成内容:**
- capability host.call retry disable 回归已补齐
- capability poll retry disable 回归已补齐
- `retry_enabled = false` 时会稳定退回单次尝试语义

### Milestone 3：文档收口

**状态:** 已完成。

**已完成内容:**
- ABI 文档已补充 capability retry disable 开关
- 计划文档与 delivery note 已补齐

## 4. 非目标范围

- 更细粒度的 `retry_poll_enabled` / `retry_call_enabled` 独立开关
- capability 之间的共享 retry budget 协调
- 与 circuit breaker / concurrency policy 的联动
