# IAVM 0.1.11 HTTP Retry Method Guard Plan

**Goal:** 在 `0.1.10` 已完成 Retry-After hint 之后，为 `network.http_fetch` 的 HTTP 状态码重试增加 method 级安全阀，让调用方可以把自动重试限制在更安全的请求方法上。

**当前状态:**
- `0.1.10` 已完整交付，`network.http_fetch` 已支持 capability 级 `retry_http_statuses` 与 `Retry-After` hint
- 当前一旦命中状态码列表，所有 method 都会进入该重试映射，缺少额外的 method 级限制

## 1. 版本目标

`0.1.11` 完成三件事：

1. **HTTP method guard**
   - 支持 `retry_http_methods`
   - 支持 `retryHTTPMethods`
   - 仅在显式配置时启用 method 过滤

2. **Host + runtime 闭环**
   - `GET` 命中 allowlist 可继续重试
   - `POST` 等未命中 method 的调用保持原始 HTTP 响应

3. **测试与文档**
   - 增加 host API 与 runtime 回归
   - 更新 ABI / 计划文档 / delivery note

## 2. Milestones

### Milestone 1：DefaultHost method guard

**状态:** 已完成。

**已完成内容:**
- `DefaultHost` 已支持 capability 级 HTTP method 过滤
- 过滤只影响 HTTP 状态码到 retryable error 的映射
- 未配置该列表时保持现有兼容行为

### Milestone 2：Host + runtime 闭环

**状态:** 已完成。

**已完成内容:**
- `GET` 命中 method guard 的回归已补齐
- `POST` 被 method guard 挡住的 runtime 回归已补齐
- 现有 `network.http_fetch` 重试链路保持可用

### Milestone 3：测试与文档收口

**状态:** 已完成。

**已完成内容:**
- host API method guard 回归已补齐
- runtime 集成回归已补齐
- 全量测试已通过

- 更复杂的 method + status policy 组合
- retry 与 circuit breaker / budget policy 的联动
