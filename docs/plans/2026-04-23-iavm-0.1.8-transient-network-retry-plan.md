# IAVM 0.1.8 Transient Network Retry Plan

**Goal:** 在 `0.1.7` 已完成 retryable host error 标记机制之后，让 `DefaultHost` 对真实网络瞬时错误主动打上 retryable 标记，使 runtime retry policy 能直接落到默认宿主实现上。

**当前状态:**
- `0.1.7` 已完整交付，runtime 已能识别宿主显式 retryable error
- `DefaultHost` 仍主要透传底层 network provider 错误，尚未把瞬时网络错误自动映射为 retryable

## 1. 版本目标

`0.1.8` 完成三件事：

1. **DefaultHost transient network marker**
   - 对瞬时 `net.Error` 自动 `MarkRetryable`
   - 仅覆盖 network I/O 路径，不扩大到 policy/参数错误

2. **网络路径覆盖**
   - `network.http_fetch`
   - `network.dial/listen/accept/send/recv`

3. **测试与文档**
   - 增加 transient network error 自动打标回归
   - 更新 ABI / 计划文档 / delivery note

## 2. Milestones

### Milestone 1：DefaultHost 网络错误映射

**状态:** 已完成。

**已完成内容:**
- `DefaultHost` 已在 network I/O 路径上识别瞬时 `net.Error`
- timeout / temporary 网络错误会自动转成 retryable
- 非网络瞬时错误不会被误标

### Milestone 2：关键路径覆盖

**状态:** 已完成。

**已完成内容:**
- `network.http_fetch`
- `network.dial`
- `network.listen`
- `network.accept`
- `network.send`
- `network.recv`

### Milestone 3：测试与文档收口

**状态:** 已完成。

**已完成内容:**
- transient network error 自动打标回归已补齐
- policy/参数错误保持非 retryable 的回归已补齐
- host API 与全量测试已通过
