# IAVM 0.0.7 Async Platform Plan

**Goal:** 在 `0.0.6` 已建立类继承与最小资源模型之后，把 IAVM 从“不识别 async”推进到“具备可验证的 async/platform 基线”，优先完成 Promise/await 的同步闭环，再逐步扩展到真正的 suspend/resume 与 network handle。

**当前状态:**
- `0.0.6` 已完整交付，`extends/super`、FS handle 与 `run-iavm --cap-config` 已稳定
- `ialang` 前端与编译器已经具备 `async function` / `await` 语法与字节码
- IAVM 此前会把 `OpAwait` lowering 成 `Nop`，导致 async 示例无法运行
- 本计划的第一步已经完成：IAVM 已保留 `OpAwait`，async function 会返回已完成 Promise，`await` 可解包 Promise 或透传普通值

## 1. 版本目标

`0.0.7` 的重点不是一次做完整异步平台，而是先建立三层递进：

1. **Promise / await 同步基线**
   - IAVM 识别 `async function`
   - async 调用返回 Promise 值
   - `await` 不再被忽略

2. **最小 suspend/resume 设计稿与实现起点**
   - 明确 pending promise / poll-result / wakeup 的 VM 语义
   - 为后续真正异步执行保留结构

3. **network handle 与 poll 语义扩展**
   - 在 `0.0.6` 文件句柄基础上，为 network handle 做准备
   - 逐步从同步 ready-result 过渡到更真实的 poll/wakeup 模型

## 2. Milestones

### Milestone 1：Promise / await 同步基线

**状态:** 已完成。

**已完成内容:**
- `OpAwait` 已进入 IAVM opcode / verifier / lowering / runtime
- `module.Function` 已携带 `Async` 元数据并参与二进制编解码
- async function 调用会返回已完成 Promise
- `await` 会解包已完成 Promise，普通值按原样透传
- `async_loop.ia` 已进入 CLI 回归矩阵

**验收标准:**
- `await` 不再在 lowering 中丢失
- `async function` 返回 Promise 值
- `async_loop.ia` 可通过 build / verify / run

### Milestone 2：Pending Promise 与 suspend/resume 骨架

**状态:** 进行中。

**已完成内容:**
- Promise 已区分 `pending` / `resolved` / `rejected`
- `await pending` 会返回明确的 `ErrPromisePending`
- VM 已记录最小 `Suspension` 状态，包含 reason / await value / frame depth
- `await rejected` 已进入单独错误路径

**下一步:**
- 在当前 `Suspension` 基础上设计 continuation / resume API
- 把 pending promise 与 host poll / wakeup 建立关联

### Milestone 3：Poll 驱动的 async 资源语义

**状态:** 进行中。

**已完成内容:**
- `OpHostPoll` 已升级为返回 Promise 值，而不是直接返回 poll 对象
- `await host.poll(handle)` 现在可进入 pending suspension
- `ResumeSuspension()` 已可再次执行 `Host.Poll(handle)`，done 后继续解释执行

**下一步:**
- 为 `network.dial/listen/send/recv` 继续补上 VM / smoke 路径
- 明确 poll / wakeup / backpressure 的文档约定

## 3. 发布前验证

当前阶段至少应保持以下命令稳定：

```bash
go test ./iavm/... ./ialang/...
```

关键回归：

```bash
go test ./iavm/pkg/runtime
go test ./ialang/cmd/ialang -run TestRunCLIAsyncLoopExampleRegression
```

## 4. 风险与边界

| 风险 | 影响 | 处理策略 |
|------|------|----------|
| 过早追求完整 suspend/resume | 范围迅速膨胀 | 先做同步 Promise 基线，再进入 pending 语义 |
| Promise 值类型破坏现有值语义 | runtime 回归 | 先把 Promise 限定在 async/await 路径，不扩散到 host ABI |
| network handle 过早进入实现 | 与 async 基线相互牵制 | 先明确 ABI 与 poll 设计，再逐步落代码 |
