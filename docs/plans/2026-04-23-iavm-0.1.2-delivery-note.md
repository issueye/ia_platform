# IAVM 0.1.2 Delivery Note

**Version:** `0.1.2`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.2` 聚焦“capability timeout profile”，在 `0.1.1` operation timeout policy 基础上完成了以下目标：

- capability `Config.host_timeout_ms` 已可覆盖默认 `HostTimeout`
- capability `Config.wait_timeout_ms` 已可覆盖默认 `WaitTimeout`
- pending promise 会保留触发时的 capability timeout profile，并在恢复路径中继续生效
- runtime 已明确 capability profile 与默认 timeout option 的优先级

## 2. 主要交付物

### 2.1 代码

- capability timeout profile 读取逻辑
- pending promise 中的 timeout profile 透传
- acquire/call/poll/wait 恢复路径上的 capability profile 生效

### 2.2 测试

- capability `host_timeout_ms` 覆盖默认 host timeout 回归
- capability `wait_timeout_ms` 覆盖默认 wait timeout 回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

## 4. 已知边界

以下内容仍不属于 `0.1.2` 的完整目标范围，应进入后续版本：

- capability profile 的动态热更新
- 更复杂的 profile 继承机制
- timeout profile 与 retry/backoff 的组合策略

## 5. 版本结论

`0.1.2` 已让 IAVM 的 timeout policy 从“全局默认”进一步走向“capability 级差异化控制”，为后续更细粒度的策略系统打下基础。
