# IAVM 0.0.6 Delivery Note

**Version:** `0.0.6`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.0.6` 聚焦“语义扩展 + 最小资源模型”，在 `0.0.5` Host ABI 收敛基础上完成了以下目标：

- `extends` / `super()` / `super.method()` 进入稳定范围
- `DefaultHost` 已支持 `fs.open/read/write/seek/close` 句柄生命周期
- `OpHostPoll` 已具备最小同步可验证语义
- `run-iavm --cap-config <file.toml>` 已可把 capability TOML 同时映射到：
  - module capability `AcquireRequest.Config`
  - 本地 FS preopen
  - Network HTTP policy

## 2. 主要交付物

### 2.1 代码

- class inheritance lowering/runtime 支持
- file handle table 与 `OpHostPoll` 最小闭环
- FS handle operation typed adapter
- CLI capability config 文件装载与 host 构建

### 2.2 测试

- runtime `super` / handle / `AcquireRequest.Config` 回归
- integration FS handle 端到端回归
- CLI `inheritance.ia` 回归
- CLI `run-iavm --cap-config` 本地 preopen 回归

### 2.3 文档

- `docs/iavm/abi-capabilities.md`
- `docs/plans/2026-04-23-iavm-0.0.6-semantic-expansion-plan.md`
- `ialang/README.md`

## 3. 发布前验证

推荐使用以下命令作为 `0.0.6` release gate：

```bash
go test ./iacommon/... ./iavm/... ./ialang/...
```

## 4. 非目标范围

以下内容不属于 `0.0.6` 交付范围，应进入后续版本：

- Promise / await 的完整挂起恢复平台化
- socket/listen 全矩阵 network handle 模型
- 更完整的 poll / wakeup / backpressure 语义
- capability config 的二进制模块序列化格式

## 5. 后续版本建议

`0.0.7` 建议转入“async/platform 深化”阶段，优先顺序：

1. Promise / await 平台化
2. network handle 资源模型
3. 更完整的 capability profile / config 管理
