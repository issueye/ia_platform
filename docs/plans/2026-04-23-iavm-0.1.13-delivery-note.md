# IAVM 0.1.13 Delivery Note

**Version:** `0.1.13`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.13` 聚焦“default safe HTTP methods”，在 `0.1.12` HTTP status class retry policy 基础上完成了以下目标：

- `network.http_fetch` 已支持 capability 级 `retry_http_default_safe_methods`
- capability 可以在不手写 method 列表的情况下启用标准幂等方法默认集
- 显式 `retry_http_methods` 仍然优先，保持已有配置语义稳定
- runtime allowlisted `host.call` 已可直接消费这一默认方法集能力

## 2. 主要交付物

### 2.1 代码

- `retry_http_default_safe_methods` / `retryHTTPDefaultSafeMethods` 解析
- `DefaultHost` 默认安全方法集判定逻辑
- runtime + DefaultHost 的默认方法集集成回归

### 2.2 测试

- host API default safe methods 回归
- runtime default safe methods 集成回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

## 4. 已知边界

以下内容仍不属于 `0.1.13` 的完整目标范围，应进入后续版本：

- 默认自动启用该开关
- 对 PATCH/POST 等方法做更复杂的幂等推断
- retry 与 circuit breaker / budget policy 的联动

## 5. 版本结论

`0.1.13` 已让 IAVM 的 HTTP 重试策略从“只能显式列 method”推进到“可选标准幂等方法默认集”，在保持配置可控的前提下进一步降低了使用门槛。
