# IAVM 0.1.17 Retry Max Elapsed Plan

**Goal:** 为 IAVM runtime 的 host retry 增加总耗时预算护栏，避免 capability 配置了较大 backoff 后，将单次 host operation 的尾延迟无界拉长。

**当前状态:**
- `0.1.16` 已完整交付，HTTP retry policy 已具备 method 维度 inclusion/exclusion 组合能力
- 当前 runtime retry 只有次数、backoff、max-backoff、multiplier、jitter 等控制项，缺少总耗时 budget

## 1. 版本目标

`0.1.17` 完成三件事：

1. **Retry elapsed budget 配置**
   - 支持 `retry_max_elapsed_ms`
   - 支持 `retryMaxElapsedMS`
   - 预算耗尽时停止后续重试并返回最后一次 retryable error

2. **Call / poll / pending promise 闭环**
   - `host.call`、`host.poll`、`wait` 共用同一 elapsed budget 规则
   - pending promise 挂起后恢复时保留 capability 级 elapsed budget 配置

3. **测试与文档**
   - 增加 runtime 单测与 capability 集成回归
   - 更新 ABI / 计划文档 / delivery note

## 2. Milestones

### Milestone 1：Runtime elapsed budget

**状态:** 已完成。

**已完成内容:**
- runtime options 与 capability profile 已支持 retry elapsed budget
- retry 循环会在预算不足以继续等待下一次 backoff 时提前停止
- 该护栏同时适用于 call-like 与 poll-like 路径

### Milestone 2：Pending promise 闭环

**状态:** 已完成。

**已完成内容:**
- pending promise 已保存 retry elapsed budget 配置
- 恢复挂起 promise 时继续沿用原有 budget 规则
- capability 覆盖项不会在 suspend/resume 间丢失

### Milestone 3：测试与文档收口

**状态:** 已完成。

**已完成内容:**
- runtime elapsed budget 单测已补齐
- capability host.call budget 集成回归已补齐
- 全量测试已通过

- 多 capability 之间的全局 retry 预算协调
- circuit breaker / concurrency limiter 与 elapsed budget 联动
