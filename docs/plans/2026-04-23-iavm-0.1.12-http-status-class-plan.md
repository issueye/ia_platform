# IAVM 0.1.12 HTTP Status Class Plan

**Goal:** 在 `0.1.11` 已完成 HTTP retry method guard 之后，为 `network.http_fetch` 增加 status class 级配置，让 capability 可以更简洁地声明 `5xx` 这类可重试状态族。

**当前状态:**
- `0.1.11` 已完整交付，`network.http_fetch` 已支持显式状态码列表、Retry-After hint 与 method guard
- 当前若想覆盖一整个状态族，仍需要把 500/502/503/504 等状态逐个列出

## 1. 版本目标

`0.1.12` 完成三件事：

1. **HTTP status class 配置**
   - 支持 `retry_http_status_classes`
   - 支持 `retryHTTPStatusClasses`
   - 与现有 status list / method guard 共同生效

2. **Host + runtime 闭环**
   - `5xx` 类错误可直接触发 retryable 映射
   - runtime allowlisted `host.call` 可消费该状态族配置

3. **测试与文档**
   - 增加 host API 与 runtime 回归
   - 更新 ABI / 计划文档 / delivery note

## 2. Milestones

### Milestone 1：DefaultHost status class 支持

**状态:** 已完成。

**已完成内容:**
- `DefaultHost` 已支持 capability 级 status class 解析
- status class 与精确 status list 可并存
- method guard 仍可继续限制这一路径

### Milestone 2：Host + runtime 闭环

**状态:** 已完成。

**已完成内容:**
- `5xx` 命中 class 配置的 host API 回归已补齐
- runtime + DefaultHost 的 `5xx -> retry -> 200` 回归已补齐
- 不匹配 class 的状态保持原始 HTTP 响应

### Milestone 3：测试与文档收口

**状态:** 已完成。

**已完成内容:**
- host API status class 回归已补齐
- runtime 集成回归已补齐
- 全量测试已通过
