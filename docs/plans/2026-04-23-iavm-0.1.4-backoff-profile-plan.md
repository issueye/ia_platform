# IAVM 0.1.4 Backoff Profile Plan

**Goal:** 在 `0.1.3` 已建立最小 retry/backoff 之后，为 IAVM 增加可配置的退避曲线，让 backoff 不再只能是固定间隔。

**当前状态:**
- `0.1.3` 已完整交付，poll/wait 已支持基础 retry/backoff
- 当前 backoff 仍是固定值，缺少 multiplier 与 max cap 控制

## 1. 版本目标

`0.1.4` 先完成三件事：

1. **全局 backoff 曲线**
   - 支持 `RetryMultiplier`
   - 支持 `RetryMaxBackoff`

2. **Capability backoff profile**
   - capability config 可覆盖默认 multiplier / max

3. **测试与文档**
   - 增加 backoff 计算与 profile 覆盖回归
   - 更新 ABI / 计划文档

## 2. Milestones

### Milestone 1：Backoff 曲线

**状态:** 进行中。

**目标:**
- retry backoff 支持 multiplier
- retry backoff 支持 max cap

### Milestone 2：Capability backoff profile

**状态:** 进行中。

**目标:**
- capability config 覆盖 multiplier / max
- promise 恢复路径沿用该 profile

### Milestone 3：文档与交付收口

**状态:** 待开始。

**目标:**
- 更新 ABI / delivery 文档
- 评估 `0.1.4` 完成度与边界

## 3. 发布前验证

```bash
go test ./iacommon/... ./iavm/... ./ialang/...
```

## 4. 非目标范围

- 随机 jitter
- 更复杂的 backoff 算法族
- backoff 与 circuit breaker 的联动策略
