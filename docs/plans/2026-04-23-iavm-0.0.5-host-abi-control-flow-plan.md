# IAVM 0.0.5 Host ABI & Control Flow Plan

**Goal:** 在 `0.0.4` 已完成 class/closure/runtime 基线的基础上，继续收敛 IAVM 的平台语义，优先完成 Host ABI 参数传递稳定化、finally 回归固化，以及为下一阶段的 typed adapter / class 继承打下清晰边界。

**当前状态:**
- `OpHostCall` 已支持从栈上传递参数对象给 host capability
- `fs.read_file` / `network.http_fetch` 等 operation 已可接收结构化参数
- `try/finally`、`try/catch/finally` 关键路径已通过 CLI 与 runtime 回归测试锁定
- 全量测试 `go test ./iacommon/... ./iavm/... ./ialang/...` 当前通过

**核心交付物:**
- Host ABI 参数传递约定文档化并稳定
- finally 语义具备持续回归保护
- 为 typed request/response adapter 留出明确实现入口
- 为 `extends` / `super` 和更完整 control-flow 语义提供下一阶段计划

**原则:** 0.0.5 优先做“平台契约收敛”，少做大而散的语言扩张；凡是进入 ABI 或 smoke matrix 的行为，都必须有测试与文档双重落点。

---

## 1. 0.0.4 之后的基线

`0.0.4` 已形成以下稳定基础：

- 对象属性访问、位运算 coercion、闭包 upvalue、基本 class/this 绑定已经完成
- IAVM smoke matrix 已覆盖 11 个稳定示例
- `class.ia`、`data.ia`、`bitwise.ia`、`closure.ia` 已纳入 CLI 回归测试
- 编码/解码/verify/runtime 的主路径均已进入全量测试

这意味着 `0.0.5` 不再需要回头补 0.0.4 的主功能，而应把平台边界继续收窄。

---

## 2. 0.0.5 问题拆分

### 2.1 已推进项

1. **Host ABI 参数传递**
   - `OpHostCall` 不再只传 operation name
   - 当 `inst.A == 1` 且参数为对象时，runtime 会直接把对象转换为 `CallRequest.Args`
   - 否则会同时提供 `args` 数组和 `arg0` / `arg1` / ... 的位置参数镜像

2. **finally 回归固化**
   - runtime 单测已覆盖 finally 后 rethrow 路径
   - CLI 回归已覆盖 `try_catch.ia` 中 `finally1` / `finally2` / `finally3`

### 2.2 仍待完成项

1. **Host ABI typed adapter**
   - 当前 `map[string]any` 仍是临时承载层
   - 已完成 FS / Network 的第一阶段 typed request/response adapter

2. **capability 声明与 operation 一致性**
   - 已在 verifier 中补上对静态 `OpHostCall` operation 名称的 capability 匹配校验

3. **class 继承**
   - `extends` / `super` 尚未进入 IAVM 稳定范围

4. **更系统的 control-flow 验收**
   - `finally` 关键路径已被覆盖，但仍缺更完整的矩阵化说明

---

## 3. 0.0.5 目标范围

### 3.1 必须完成

1. **Host ABI 参数协议稳定化**
   - 固化 `OpHostCall` 的栈约定与参数编码规则
   - 为 runtime、integration、CLI 保持一致语义

2. **finally 语义持续可回归**
   - 保证同步异常、嵌套异常、finally 后 rethrow 三类路径持续受保护

3. **文档同步**
   - README 与 ABI 文档明确说明当前 0.0.5 已完成项与后续项

### 3.2 建议完成

1. **FS / Network typed adapter 设计稿**
   - 即便不一次性全部实现，也至少明确 adapter 入口与边界

2. **更明确的 smoke / regression 分类**
   - 把 Host ABI 和 control-flow 相关回归归入固定分组

### 3.3 非目标范围

以下内容不纳入本轮 `0.0.5` 最小闭环：

- Promise / await 在 IAVM 中的完整平台化支持
- 外部模块系统
- JIT / AOT / native backend
- 完整 class 继承实现
- Host handle table 与 `OpHostPoll` 的完整异步资源模型

## 4. 交付路径

### Milestone 1：Host ABI 参数传递闭环

**目标:** 让 Host ABI 真正能接收业务参数，而不是停留在 operation-level 占位。

**当前状态:** 已完成。

**已完成内容:**
- `OpHostCall` 支持 `inst.A` 个参数
- 单对象参数直接映射为 `CallRequest.Args`
- 非对象参数按位置参数进行兼容编码
- 已补 runtime / integration 测试

**验收标准:**
- Host mock 能收到 `path`、`url`、`headers` 等结构化参数
- encode/decode/verify 后参数传递行为不变

### Milestone 2：finally 语义回归闭环

**目标:** 不追求新增 finally opcode，而是锁定当前编译策略对应的 runtime 行为。

**当前状态:** 已完成回归固化。

**已完成内容:**
- runtime 测试覆盖 finally 后 rethrow
- CLI 回归覆盖 `try_catch.ia`

**验收标准:**
- `finally1` / `finally2` / `finally3` 稳定出现
- `flag: 2` 稳定成立
- 异常路径不会绕过 finally 补偿块

### Milestone 3：typed adapter 设计与分阶段接入

**目标:** 让 Host ABI 从“能传”走向“易维护”。

**当前状态:** FS adapter 与 Network HTTP adapter 已完成第一阶段接入。

**任务:**
1. 为 FS operation 设计并接入 typed request/response adapter
2. 为 Network HTTP operation 设计 typed request/response adapter
3. 约束 `map[string]any` 只作为过渡层

**验收标准:**
- 至少有一类 capability 拥有清晰 adapter 入口
- 文档中说明兼容策略与演进规则

### Milestone 4：0.0.5 发布前验证

**目标:** 保证 0.0.5 的平台收敛项不会破坏 0.0.4 主路径。

**必跑命令:**

```bash
go test ./iacommon/...
go test ./iavm/...
go test ./ialang/...
```

**关键回归:**

```bash
go test ./iavm/pkg/runtime -run TestCapability_
go test ./iavm/pkg/runtime -run TestInterpret_TryFinallyRethrow
go test ./iavm/pkg/integration -run TestFullPipeline_WithHostCapability
go test ./ialang/cmd/ialang -run TestRunCLITryCatchExampleRegression
```

**验收标准:**
- 全量 Go 测试通过
- Host ABI 参数回归通过
- try/finally / try/catch/finally 回归通过
- 无 0.0.4 smoke regression

## 6. 风险与处理策略

| 风险 | 影响 | 处理策略 |
|------|------|----------|
| Host ABI 继续依赖 `map[string]any` 扩散 | 后续 SDK / verifier 难以维护 | 用 typed adapter 把边界收住 |
| finally 语义只靠少量样例覆盖 | 后续异常路径改动容易回归 | 保留 runtime + CLI 双回归 |
| 0.0.5 范围继续膨胀到 async/class 继承 | 交付节奏失焦 | 坚持“契约收敛优先” |

## 8. 0.0.5 之后的方向

当 `0.0.5` 收敛项完成后，再进入更明显的语义扩展阶段：

1. **class 继承**
   - `extends` / `super`
   - 原型链与父类方法分发

2. **Host ABI typed adapter 全面化**
   - FS / Network request/response 类型化
   - 更稳定的 verifier 与 CLI 配置映射

3. **Host handle table / poll 模型**
   - `fs.open/read/write/seek/close`
   - `network.dial/listen/send/recv`
   - `OpHostPoll` 的真正异步语义

4. **更完整的 async/await 平台化**
   - Promise 值类型
   - await 挂起与恢复
