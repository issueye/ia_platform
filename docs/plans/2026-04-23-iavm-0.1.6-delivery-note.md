# IAVM 0.1.6 Delivery Note

**Version:** `0.1.6`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.6` 聚焦“safe host.call retry profile”，在 `0.1.5` jitter policy 基础上完成了以下目标：

- `runtime.Options.RetryCallOps` 已进入 runtime 配置面
- capability config 已可通过 `retry_call_ops` 覆盖默认 allowlist
- `host.call` 仅对 allowlist 中的 operation 启用 timeout retry
- allowlist 命中的 `host.call` 会复用已有 retry/backoff/jitter profile

## 2. 主要交付物

### 2.1 代码

- `runtime.Options.RetryCallOps`
- `host.call` retry allowlist 判定
- capability retry allowlist 解析与覆盖
- `host.call` timeout retry helper

### 2.2 测试

- 默认不重试回归
- 全局 allowlist 重试回归
- capability allowlist 重试回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

### 2.3 文档

- `docs/iavm/abi-capabilities.md`
- `docs/plans/2026-04-23-iavm-0.1.6-safe-host-call-retry-plan.md`
- 本交付说明

## 3. 发布前验证

`0.1.6` release gate:

```bash
go test ./iavm/pkg/runtime

```

## 4. 已知边界

以下内容仍不属于 `0.1.6` 的完整目标范围，应进入后续版本：

- 对所有 `host.call` 自动重试
- 按 error class 区分 retry policy
- retry 与 circuit breaker / budget policy 的联动

## 5. 版本结论

`0.1.6` 已让 IAVM 的 retry profile 从 `poll/wait` 扩展到“显式安全”的 `host.call` 路径，同时保持默认语义保守稳定。
