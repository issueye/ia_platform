# IAVM 0.1.11 Delivery Note

**Version:** `0.1.11`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.11` 聚焦“HTTP retry method guard”，在 `0.1.10` Retry-After backoff hint 基础上完成了以下目标：

- `network.http_fetch` 已支持 capability 级 `retry_http_methods`
- 配置该列表后，只有命中的 method 才允许 HTTP 状态码映射为 retryable error
- 未配置该列表时，仍保持 `0.1.10` 的兼容行为
- runtime allowlisted `host.call` 已可直接消费这一 method guard 约束

## 2. 主要交付物

### 2.1 代码

- `retry_http_methods` / `retryHTTPMethods` 解析
- `DefaultHost` method guard 逻辑
- runtime + DefaultHost 的 method guard 集成回归

### 2.2 测试

- host API GET/POST method guard 回归
- runtime disallowed method 回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

## 4. 已知边界

以下内容仍不属于 `0.1.11` 的完整目标范围，应进入后续版本：

- 默认自动推断 HTTP 方法是否幂等
- 更复杂的 method + status policy 组合
- retry 与 circuit breaker / budget policy 的联动

## 5. 版本结论

`0.1.11` 已让 IAVM 的 HTTP 重试策略从“只看状态码和服务端提示”推进到“状态码 + 服务端提示 + method 安全阀”，让默认宿主下的自动重试边界更可控。
