# IAVM 0.1.16 Delivery Note

**Version:** `0.1.16`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.16` 聚焦“HTTP method exclusion”，在 `0.1.15` HTTP status-class exclusion 基础上完成了以下目标：

- `network.http_fetch` 已支持 capability 级 `retry_http_excluded_methods`
- capability 现在可以从 method 重试策略中显式排除具体 HTTP method
- 被排除 method 会保持原始 HTTP 响应语义
- runtime allowlisted `host.call` 已可直接消费这一例外规则

## 2. 主要交付物

### 2.1 代码

- `retry_http_excluded_methods` / `retryHTTPExcludedMethods` 解析
- `DefaultHost` method exclusion 优先级逻辑
- runtime + DefaultHost 的 exclusion method 集成回归

### 2.2 测试

- host API method exclusion 回归
- runtime exclusion method 不重试回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

- `docs/iavm/abi-capabilities.md`

## 4. 已知边界

以下内容仍不属于 `0.1.16` 的完整目标范围，应进入后续版本：

- 更复杂的 method policy 优先级 DSL
- 基于幂等语义自动推断 method exclusion
- retry 与 budget / circuit breaker policy 的联动

## 5. 版本结论

`0.1.16` 已让 IAVM 的 HTTP 重试策略在 method 维度同时具备 inclusion 与 exclusion 能力，使能力配置在真实流量治理场景里更易组合和收敛。
