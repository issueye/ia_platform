# IAVM 0.1.17 Delivery Note

**Version:** `0.1.17`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.17` 聚焦“retry max elapsed budget”，为 runtime host retry 增加了总耗时护栏：

- capability 现已支持 `retry_max_elapsed_ms`
- retry budget 耗尽时，runtime 会停止后续重试并返回最后一次 retryable error
- `host.call`、`host.poll`、`wait` 与 pending promise 恢复流程均共享这一规则

## 2. 主要交付物

### 2.1 代码

- runtime retry elapsed budget 解析与执行逻辑
- pending promise retry profile 的 elapsed budget 持久化
- capability 级 host.call budget 集成回归

### 2.2 测试

- runtime call/poll elapsed budget 单测
- pending promise profile 持久化回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

## 4. 已知边界

以下内容仍不属于 `0.1.17` 的完整目标范围，应进入后续版本：

- 更复杂的 retry budget 分层与配额共享
- 多 capability 之间的全局 retry 预算协调
- circuit breaker / concurrency limiter 与 elapsed budget 联动

## 5. 版本结论

`0.1.17` 让 IAVM 的 host retry 从“只控制次数与间隔”进一步升级为“还能限制整条重试链路的总耗时”，更适合生产场景里的尾延迟治理。
