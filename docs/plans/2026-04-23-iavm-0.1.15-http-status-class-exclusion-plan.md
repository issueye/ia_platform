# IAVM 0.1.15 HTTP Status Class Exclusion Plan

**Goal:** 在 `0.1.14` 已完成 HTTP status exclusion 之后，为 `network.http_fetch` 增加状态族级排除例外，让状态族策略可以被更粗粒度地显式关掉。

**当前状态:**
- `0.1.14` 已完整交付，`network.http_fetch` 已支持具体状态码 exclusion
- 当前如果想从 `5xx` 这类策略里排除整类状态，仍只能逐个状态码枚举

## 1. 版本目标

`0.1.15` 完成三件事：

1. **HTTP status-class exclusion 配置**
   - 支持 `retry_http_excluded_status_classes`
   - 支持 `retryHTTPExcludedStatusClasses`
   - 命中时优先于状态码列表、状态族和默认方法集

2. **Host + runtime 闭环**
   - 被排除的状态族保持原始 HTTP 响应
   - runtime allowlisted `host.call` 不会对排除状态族触发重试

3. **测试与文档**
   - 增加 host API 与 runtime 回归
   - 更新 ABI / 计划文档 / delivery note

## 2. Milestones

### Milestone 1：DefaultHost status-class exclusion

**状态:** 已完成。

**已完成内容:**
- `DefaultHost` 已支持 capability 级状态族排除配置
- 状态族排除优先级高于状态码列表和状态族命中配置
- 该规则可与 method/default-safe-methods 组合使用

### Milestone 2：Host + runtime 闭环

**状态:** 已完成。

**已完成内容:**
- host API 的状态族排除回归已补齐
- runtime + DefaultHost 的状态族排除不重试回归已补齐
- 被排除状态族保持原始 HTTP 响应与单次尝试

### Milestone 3：测试与文档收口

**状态:** 已完成。

**已完成内容:**
- host API status-class exclusion 回归已补齐
- runtime 集成回归已补齐
- 全量测试已通过
