# IAVM 0.1.23 Delivery Note

**Version:** `0.1.23`  
**Date:** `2026-04-24`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.23` 聚焦“capability poll-retry disable”，为 capability 增加了仅关闭 `poll/wait` 自动重试的细粒度开关：

- capability 现已支持 `retry_poll_enabled`
- `retry_poll_enabled = false` 时，该 capability 的 `poll/wait` 不再自动重试
- 同一 capability 的 allowlisted `host.call` retry 行为保持不变

## 2. 主要交付物

### 2.1 代码

- capability poll-retry disable 配置解析
- pending promise poll-retry 开关持久化
- poll disable 与 host.call retry 保持生效回归测试

### 2.2 测试

- capability poll retry disable 回归
- capability host.call retry 保持生效回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

## 4. 已知边界

以下内容仍不属于 `0.1.23` 的完整目标范围，应进入后续版本：

- 更细粒度的 `retry_wait_enabled` 独立开关
- 对 `poll` 与 `wait` 分别设置不同 retry 配置
- 与 circuit breaker / concurrency policy 的联动

## 5. 版本结论

`0.1.23` 让 IAVM 的 capability retry 开关进一步细化到“可单独关闭 poll/wait retry”，更适合异步等待和调用重放风险需要分开治理的场景。
