# IAVM 0.1.15 Delivery Note

**Version:** `0.1.15`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.15` 聚焦“HTTP status class exclusion”，在 `0.1.14` HTTP status exclusion 基础上完成了以下目标：

- `network.http_fetch` 已支持 capability 级 `retry_http_excluded_status_classes`
- capability 现在可以从状态族重试策略中显式排除整个状态族
- 被排除状态族会保持原始 HTTP 响应语义
- runtime allowlisted `host.call` 已可直接消费这一例外规则

## 2. 主要交付物

### 2.1 代码

- `retry_http_excluded_status_classes` / `retryHTTPExcludedStatusClasses` 解析
- `DefaultHost` status-class exclusion 优先级逻辑
- runtime + DefaultHost 的 status-class exclusion 集成回归

### 2.2 测试

- host API status-class exclusion 回归
- runtime exclusion class 不重试回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

## 4. 已知边界

以下内容仍不属于 `0.1.15` 的完整目标范围，应进入后续版本：

- 更复杂的 inclusion/exclusion 优先级 DSL
- 自动推断哪些状态族应该默认排除
- retry 与 circuit breaker / budget policy 的联动

## 5. 版本结论

`0.1.15` 已让 IAVM 的 HTTP 重试策略从“能排除单个状态码”推进到“也能排除整个状态族”，让组合策略在真实场景里更容易维护。
