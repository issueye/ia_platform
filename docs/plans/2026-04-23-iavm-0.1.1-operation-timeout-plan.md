# IAVM 0.1.1 Operation Timeout Plan

**Goal:** 在 `0.1.0` 已建立 timeout / cancellation 基线后，为 IAVM 增加单次 host operation 级别的超时策略，避免单个 acquire/call/poll/wait 无限阻塞。

**当前状态:**
- `0.1.0` 已完整交付，settled 执行已具备总 deadline 与 cancellation 语义
- 当前 timeout 仍主要作用于整个执行过程，缺少单次 host operation 的独立超时边界

## 1. 版本目标

`0.1.1` 先完成三件事：

1. **Host 操作级超时**
   - `HostTimeout` 作用于 acquire/call/poll

2. **Wait 操作级超时**
   - `WaitTimeout` 作用于 `Wait(handle)`

3. **测试与文档**
   - 增加 operation timeout 回归
   - 更新 ABI / 计划文档

## 2. Milestones

### Milestone 1：Host 操作超时

**状态:** 进行中。

**目标:**
- 为 acquire/call/poll 建立单次超时上下文
- 恢复路径中的 poll 也纳入操作超时

### Milestone 2：Wait 操作超时

**状态:** 进行中。

**目标:**
- `Wait(handle)` 使用独立 wait timeout
- 为 blocked wait 增加 deadline exceeded 回归

### Milestone 3：文档与交付收口

**状态:** 待开始。

**目标:**
- 更新 ABI / delivery 文档
- 评估 `0.1.1` 完成度与边界

## 3. 发布前验证

```bash
go test ./iacommon/... ./iavm/... ./ialang/...
```

## 4. 非目标范围

- per-capability 动态 timeout profile
- 优先级感知的 timeout 策略
- 更高级的 retry/backoff 机制
