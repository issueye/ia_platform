# IAVM 0.0.9 Scheduler Hardening Plan

**Goal:** 在 `0.0.8` 已完成 wait/wakeup 最小平台化后，把等待与恢复逻辑进一步收敛到 runtime 内部，并为后续 timeout/fairness/多 suspension 调度做好统一入口。

**当前状态:**
- `0.0.8` 已完整交付，VM 已具备 `WaitSuspension(ctx)` 与 CLI 自动 wait/resume
- `0.0.9` 已把自动恢复主循环收敛进 runtime
- 对未实现 `Waiter` 的宿主，runtime 已具备轮询 fallback 等待策略

## 1. 版本目标

`0.0.9` 先完成三件事：

1. **统一运行入口**
   - VM 提供 `RunUntilSettled(ctx)` 统一收敛 pending/resume 主循环

2. **Wait fallback 策略**
   - 对无 `Waiter` 宿主，runtime 退回到可配置轮询等待

3. **后续调度骨架**
   - 为 timeout、fairness、多 suspension 调度预留配置位与结构边界

## 2. Milestones

### Milestone 1：RunUntilSettled 统一入口

**状态:** 已完成。

**已完成内容:**
- runtime 已提供 `RunUntilSettled(ctx)`
- CLI 已切换到 runtime API
- runtime / CLI 回归已覆盖 pending 自动恢复路径

### Milestone 2：无 Waiter 宿主兜底

**状态:** 已完成。

**已完成内容:**
- `WaitSuspension()` 已在无 `Waiter` 时回退到 `Host.Poll`
- runtime `Options` 已引入最小等待间隔配置
- runtime / CLI 回归已覆盖 fallback 路径

### Milestone 3：文档与交付收口

**状态:** 已完成。

**已完成内容:**
- 调度语义文档已更新
- `0.0.9` delivery note 已生成
- 当前版本完成度与边界已明确

## 4. 非目标范围

- 真实 OS 事件循环
