# IAVM 0.1.7 Delivery Note

**Version:** `0.1.7`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.7` 聚焦“retryable host error”，在 `0.1.6` safe `host.call` retry profile 基础上完成了以下目标：

- `iacommon/pkg/host/api` 已支持显式 retryable error marker
- runtime retry classifier 已支持 retryable host error
- `poll/wait` 与 allowlisted `host.call` 都可复用这一错误分类能力
- 默认保守边界保持不变：未 allowlist 的 `host.call` 仍不会自动重试

## 2. 主要交付物

### 2.1 代码

- `api.MarkRetryable(err)` / `api.IsRetryableError(err)`
- runtime retry classifier 扩展
- `poll/wait` retryable error 支持
- allowlisted `host.call` retryable error 支持

### 2.2 测试

- host API retryable marker 回归
- `poll/wait` retryable error 回归
- `host.call` retryable / non-retryable error 回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

### 2.3 文档

- `docs/iavm/abi-capabilities.md`
- `docs/plans/2026-04-23-iavm-0.1.7-retryable-host-error-plan.md`
- 本交付说明

go test ./iacommon/pkg/host/api ./iavm/pkg/runtime

## 4. 已知边界

以下内容仍不属于 `0.1.7` 的完整目标范围，应进入后续版本：

- 更细粒度的 retry error taxonomy
- 基于 error class 的独立 backoff 策略
- retry 与 circuit breaker / budget policy 的联动

## 5. 版本结论

`0.1.7` 已让 IAVM 的 retry policy 从“timeout 驱动”推进到“timeout + 宿主显式错误分类”双轨机制，为后续更精细的容错策略打下基础。
