# IAVM 0.1.5 Jitter Policy Plan

**Goal:** 在 `0.1.4` 已支持 backoff 曲线之后，为 IAVM 增加可配置的 retry jitter，让重试间隔可以在保持默认兼容的前提下引入受控随机性。

**当前状态:**
- `0.1.4` 已完整交付，retry/backoff 已支持 multiplier 与 max cap
- 当前默认 backoff 仍是确定性退避；缺少 jitter 配置与 profile 透传

## 1. 版本目标

`0.1.5` 完成三件事：

1. **全局 jitter 配置**
   - 支持 `Options.RetryJitter`
   - 默认 `0` 时保持现有行为不变

2. **Capability jitter profile**
   - capability config 支持 `retry_jitter`
   - pending promise 与恢复路径沿用触发时的 jitter profile

3. **测试与文档**
   - 增加 jitter 范围与可重复性回归
   - 更新 ABI / 计划文档 / delivery note

## 2. Milestones

### Milestone 1：Jitter runtime 支持

**状态:** 已完成。

**已完成内容:**
- runtime 已支持 `RetryJitter`
- jitter 仅在 backoff 大于 `0` 且 ratio 大于 `0` 时生效
- jitter 结果继续受 max backoff cap 约束

### Milestone 2：Capability profile 透传

**状态:** 已完成。

**已完成内容:**
- capability config 已可读取 `retry_jitter`
- `host.poll -> promise -> wait/resume` 全链路已透传 jitter profile
- pending promise 恢复路径会沿用保存下来的 jitter 策略

### Milestone 3：测试与文档收口

**状态:** 已完成。

**已完成内容:**
- 固定随机种子的 jitter 回归已补齐
- capability jitter profile 覆盖回归已补齐
- ABI / delivery note 已更新

## 3. 发布前验证

```bash
go test ./iavm/pkg/runtime
go test ./iacommon/... ./iavm/... ./ialang/...
```

## 4. 非目标范围

- 更复杂的 jitter 分布算法
- backoff / jitter 与 circuit breaker 的联动
- 对 `host.call` 引入自动重试
