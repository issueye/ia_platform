# IAVM 0.0.9 Delivery Note

**Version:** `0.0.9`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.0.9` 聚焦“调度硬化”，在 `0.0.8` wait/wakeup 最小平台化基础上完成了以下目标：

- runtime 提供统一 `RunUntilSettled(ctx)` 入口，收敛 pending/resume 主循环
- `WaitSuspension(ctx)` 在宿主未实现 `Waiter` 时可退回到 `Host.Poll` 轮询等待
- runtime `Options` 增加最小等待间隔配置，为后续 timeout/fairness 留出扩展位
- CLI `run-iavm` 已完全复用 runtime 调度入口

## 2. 主要交付物

### 2.1 代码

- `runtime.VM.RunUntilSettled(ctx)`
- `WaitSuspension()` 的 fallback polling 路径
- `runtime.Options.WaitInterval`
- CLI 入口从本地 helper 收敛到 runtime API

### 2.2 测试

- runtime `RunUntilSettled()` 回归
- runtime 无 `Waiter` 宿主 fallback 回归
- CLI fallback 调度回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

- 本交付说明


## 4. 已知边界

以下内容仍不属于 `0.0.9` 的完整目标范围，应进入后续版本：

- 多 suspension / 多 VM 的公平调度器
- 真正基于 OS 事件源的 wait/wakeup
- timeout、优先级、backpressure 的系统化策略

## 5. 版本结论

`0.0.9` 已把等待与恢复的主循环统一收敛到 runtime 层，并为后续更真实的 scheduler 演进打好了最小骨架。
