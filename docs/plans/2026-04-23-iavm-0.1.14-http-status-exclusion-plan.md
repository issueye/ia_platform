# IAVM 0.1.14 HTTP Status Exclusion Plan

**Goal:** 在 `0.1.13` 已完成 default safe HTTP methods 之后，为 `network.http_fetch` 增加状态码级排除例外，让状态族与默认策略可以安全地挖掉不想重试的具体状态。

**当前状态:**
- `0.1.13` 已完整交付，`network.http_fetch` 已支持 status/status-class + method/default-safe-methods 配置
- 当前一旦命中状态码或状态族策略，缺少针对单个状态码的“显式例外”能力

## 1. 版本目标

`0.1.14` 完成三件事：

1. **HTTP status exclusion 配置**
   - 支持 `retry_http_excluded_statuses`
   - 支持 `retryHTTPExcludedStatuses`
   - 命中时优先于状态码列表、状态族和默认方法集

2. **Host + runtime 闭环**
   - 被排除的状态保持原始 HTTP 响应
   - runtime allowlisted `host.call` 不会对排除状态触发重试

3. **测试与文档**
   - 增加 host API 与 runtime 回归
   - 更新 ABI / 计划文档 / delivery note

## 2. Milestones

### Milestone 1：DefaultHost status exclusion

**状态:** 已完成。

**已完成内容:**
- `DefaultHost` 已支持 capability 级状态码排除配置
- 排除状态优先级高于状态码列表和状态族配置
- 该规则与 method/default-safe-methods 配置可组合使用

### Milestone 2：Host + runtime 闭环

**状态:** 已完成。

**已完成内容:**
- host API 的排除状态回归已补齐
- runtime + DefaultHost 的排除状态不重试回归已补齐
- 被排除状态保持原始 HTTP 响应与单次尝试

### Milestone 3：测试与文档收口

**状态:** 已完成。

**已完成内容:**
- host API status exclusion 回归已补齐
- runtime 集成回归已补齐
- 全量测试已通过
