# IAVM 0.1.23 Capability Poll Retry Disable Plan

**Goal:** 为 capability 增加 `retry_poll_enabled` 细粒度开关，让单个 capability 可以只关闭 `poll/wait` 自动重试，而保留 `host.call` retry。

**当前状态:**
- `0.1.22` 已完整交付，capability 已支持 `retry_call_enabled`
- 当前若只想关闭 `poll/wait` retry、同时保留 `host.call` retry，仍只能用总开关一起关闭

## 1. 版本目标

`0.1.23` 完成三件事：

1. **Capability poll-retry disable 配置**
   - 支持 `Config.retry_poll_enabled`
   - 支持 `Config.retryPollEnabled`
   - 为 `false` 时只关闭该 capability 的 `poll/wait` 自动重试

2. **Poll/wait + host.call 闭环**
   - `poll/wait` 在 capability 级 poll-retry disable 下回退为单次尝试
   - 同一 capability 的 allowlisted `host.call` 仍可继续使用 retry/backoff profile

3. **测试与文档**
   - 增加 runtime capability poll-retry disable 回归
   - 更新 ABI / 计划文档 / delivery note

## 2. Milestones

### Milestone 1：Capability poll-retry disable

**状态:** 已完成。

**已完成内容:**
- capability profile 与 pending promise state 已支持 `retry_poll_enabled`
- 关闭时只影响 `poll/wait` retry，不影响 `host.call` retry policy
- suspend/resume 路径会保留该 capability 的 poll-retry 开关语义

### Milestone 2：运行时闭环

**状态:** 已完成。

**已完成内容:**
- capability poll retry disable 回归已补齐
- capability host.call retry 保持生效回归已补齐
- `retry_poll_enabled = false` 时语义稳定收敛到“poll 单次、call 可重试”

### Milestone 3：文档收口

**状态:** 已完成。

**已完成内容:**
- ABI 文档已补充 capability poll-retry disable 开关
- 计划文档与 delivery note 已补齐

## 4. 非目标范围

- 更细粒度的 `retry_wait_enabled` 独立开关
- 对 `poll` 与 `wait` 分别设置不同 retry 配置
- 与 circuit breaker / concurrency policy 的联动
