# IAVM 0.1.0 Timeout Cancellation Plan

**Goal:** 在 `0.0.9` 已把调度主循环统一收敛到 runtime 之后，为 IAVM 建立首个真正可验证的 timeout / cancellation 执行基线。

**当前状态:**
- `0.0.9` 已完整交付，`RunUntilSettled(ctx)` 与 fallback polling 已稳定
- runtime 的 pending/wakeup 主循环已经可控，但 timeout/cancellation 语义仍未完全贯穿到 host 调用与解释执行

## 1. 版本目标

`0.1.0` 先完成三件事：

1. **统一执行上下文**
   - runtime 内部 host 调用共用同一执行 context

2. **Timeout 基线**
   - `RunUntilSettled(ctx)` 尊重外部 context 超时
   - `Options.MaxDuration` 形成 runtime 级超时兜底

3. **Cancellation 回归**
   - runtime / CLI 增加 deadline exceeded / cancel 回归

## 2. Milestones

### Milestone 1：执行上下文收敛

**状态:** 已完成。

**已完成内容:**
- runtime 内部 host acquire/call/poll 已共享统一执行上下文
- 解释执行循环已可观察 context 终止
- settled 执行与等待恢复不再丢失外部 context

### Milestone 2：timeout/cancel 基线

**状态:** 已完成。

**已完成内容:**
- `RunUntilSettled(ctx)` 已尊重外部 timeout/cancel
- `MaxDuration` 已形成 runtime 级 deadline 兜底
- runtime / CLI 已覆盖 deadline exceeded 回归

### Milestone 3：文档与交付收口

**状态:** 进行中。

**目标:**
- 更新 ABI / delivery 文档
- 评估 `0.1.0` 完成度与边界

## 3. 发布前验证

```bash
go test ./iacommon/... ./iavm/... ./ialang/...
```

## 4. 非目标范围

- 抢占式取消
- 细粒度 per-operation timeout policy
- 多任务公平调度
