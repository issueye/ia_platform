# IAVM 0.1.19 Delivery Note

**Version:** `0.1.19`  
**Date:** `2026-04-23`  
**Status:** 已完整交付

## 1. 交付范围

`0.1.19` 聚焦“host.call prefix retry allowlist”，为 runtime 的 operation 级重试策略补上了前缀放行能力：

- runtime 现已支持 `Options.RetryCallOpPrefixes`
- capability 现已支持 `retry_call_op_prefixes`
- 命中允许前缀、且未命中 exclusion 的 `host.call` operation 现可自动重试

## 2. 主要交付物

### 2.1 代码

- runtime host.call prefix allowlist 配置解析
- exact allowlist / prefix allowlist / exact exclusion 优先级逻辑
- runtime prefix allowlist 回归测试

### 2.2 测试

- 默认 prefix allowlist retry 回归
- capability prefix allowlist + exact exclusion 冲突回归
- 全量 `go test ./iacommon/... ./iavm/... ./ialang/...` 通过

### 2.3 文档

- `docs/iavm/abi-capabilities.md`
- `docs/plans/2026-04-23-iavm-0.1.19-host-call-prefix-retry-plan.md`

`0.1.19` release gate:

```bash
go test ./iacommon/pkg/host/api ./iavm/pkg/runtime
go test ./iacommon/... ./iavm/... ./ialang/...
```

## 4. 已知边界

以下内容仍不属于 `0.1.19` 的完整目标范围，应进入后续版本：

- 更复杂的 glob / regex operation matching
- prefix 级 exclusion 规则
- 与 circuit breaker / concurrency policy 的联动

## 5. 版本结论

`0.1.19` 让 IAVM 的 host.call retry policy 从“逐条 operation 枚举”推进到“支持按 operation family 批量放行”，更适合真实 capability 面不断扩展的场景。
